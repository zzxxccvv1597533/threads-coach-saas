# 最終優化方案清單

## 討論結論

經過完整的討論和批判性審視，確認以下方向：

> **AI 感降低 + 更口語化 = 更有流量**

本報告整理所有優化項目，包含具體做法和優先順序。

---

## 一、優化項目總覽

| 優先級 | 項目 | 目標 | 預估時間 |
|--------|------|------|----------|
| **P0** | 提示詞精簡與口語化 | 降低 AI 感 40-50% | 4-6 小時 |
| **P0** | 各貼文類型專屬結構 | 避免模板感 | 3-4 小時 |
| **P1** | 選題靈感功能（簡化版） | 解決選題困難 | 6-8 小時 |
| **P1** | 輸入欄位優化 | 引導更具體的素材 | 4-6 小時 |
| **P2** | AI 詞彙替換後處理 | 進一步降低 AI 感 | 2-3 小時 |
| **P2** | CTA 結尾多樣化 | 避免結尾公式化 | 2-3 小時 |

**總計：約 21-30 小時（3-4 天）**

---

## 二、P0 優化項目詳細說明

### 項目 1：提示詞精簡與口語化

#### 問題
- 目前提示詞約 7,500 字，規則 30+ 條
- LLM 無法全部遵守，選擇性執行產生拼湊感
- 規則互相矛盾（「口語化」vs「結構化」）

#### 做法

**Step 1：精簡提示詞**
- 從 7,500 字精簡到 1,000-1,500 字
- 從 30+ 條規則精簡到 5-8 條核心規則
- 移除互相矛盾的規則

**Step 2：強調口語化**
- 核心指令：「像跟朋友聊天一樣說話」
- 禁止：「此外」「值得注意的是」「至關重要」等 AI 詞彙
- 鼓勵：「欸」「啊」「真的」「超」等口語詞

**Step 3：移除具體範例注入**
- 不再把爆款範例注入提示詞
- 改為注入「結構原則」（但不強制）

#### 修改前後對比

**修改前（約 7,500 字）**：
```
你是一位專業的 Threads 內容創作專家...
【四透鏡框架】
1. 自我透鏡：...
2. 他者透鏡：...
3. 情境透鏡：...
4. 時間透鏡：...
【爆款元素】
1. 開頭要有 Hook...
2. 中間要有故事...
3. 結尾要有 CTA...
【禁止事項】
1. 不要使用「此外」...
2. 不要使用「值得注意的是」...
... (30+ 條規則)
【範例】
以下是一篇爆款貼文範例：
「三年前，我也是那個...」
```

**修改後（約 1,000 字）**：
```
你是一位 Threads 創作者，幫用戶把想法寫成貼文。

【核心原則】
1. 像跟朋友聊天一樣說話
2. 用真實的故事和感受
3. 句子要有長有短，有節奏

【禁止詞彙】
此外、值得注意的是、至關重要、深入探討、不僅...還...

【口語詞彙（鼓勵使用）】
欸、啊、真的、超、蠻、其實、然後、就是

【結構建議（不強制）】
- 開頭：直接說重點或問問題
- 中間：說故事或分享觀點
- 結尾：可以問問題，也可以不問
```

#### 預期效果
- AI 感降低 40-50%
- 內容更口語、更自然
- 用戶滿意度提升

---

### 項目 2：各貼文類型專屬結構

#### 問題
- 目前有「通用規則」覆蓋所有類型
- 所有貼文都被強制套用「Hook + 故事 + CTA」
- 產生模板感

#### 做法

**Step 1：移除通用結構規則**
- 不再強制所有貼文都要 Hook + 故事 + CTA
- 每種類型有自己的結構指引

**Step 2：定義各類型專屬結構**

| 類型 | 結構 | 結尾 |
|------|------|------|
| **故事型** | 時間 → 衝突 → 轉折 → 啟發 | 反思（不一定要問問題） |
| **知識型** | 問題 → 重點 → 行動建議 | 行動建議 |
| **閒聊型** | 情緒 → 碎片 → 隨意 | **可以沒有結尾** |
| **提問型** | 問題 → 背景 | 問題本身 |
| **觀點型** | 觀點 → 論點 → 討論 | 討論或反問 |
| **反差型** | 預期 → 反轉 → 解釋 | 可有可無 |
| **對話型** | 問題 → 回答 → 延伸 | 延伸思考 |
| **診斷型** | 症狀 → 分析 → 建議 | 建議或鼓勵 |
| **清單型** | 主題 → 列點 → 總結 | 總結或邀請 |
| **金句型** | 金句 → 解釋 → 共鳴 | 共鳴或留白 |

**Step 3：修改提示詞組裝邏輯**
- 根據用戶選擇的貼文類型，注入對應的結構指引
- 不再注入「通用結構規則」

#### 預期效果
- 不同類型的貼文有不同的風格
- 避免「所有貼文都長一樣」的問題
- 更符合 Threads 的多元風格

## 三、P1 優化項目詳細說明

### 項目 3：選題靈感功能（簡化版）

#### 問題
- 用戶不知道要寫什麼
- 原本設計太複雜（5 個選題 × 3 個切角 × 5 個 Hook）

#### 做法（簡化版）

**設計原則**：
1. 給「1 個推薦」而不是「5 個選擇」
2. 不滿意可以「換一個」
3. 不顯示「爆紅機率」，改用「適合度」

**前端介面**：
```
┌─────────────────────────────────────────────┐
│  🎯 今日推薦選題                              │
│                                             │
│  「漲價的焦慮」                               │
│                                             │
│  💡 這個選題很適合你的風格                     │
│  📝 建議類型：故事型                          │
│  ✨ 建議開頭：「上週有個學員問我...」           │
│                                             │
│  [使用這個推薦]  [換一個]  [我有自己的想法]    │
└─────────────────────────────────────────────┘
```

**兩種入口**：
1. **我沒想法** → 顯示「今日推薦選題」
2. **我有想法** → 直接進入創作流程

**技術實作**：
- 根據用戶的 IP 地基（領域、風格）過濾選題
- 從爆款資料庫中選擇相關選題
- 用 AI 生成「建議開頭」

#### 預期效果
- 解決「不知道寫什麼」的問題
- 流程簡單，不增加用戶負擔
- 符合平台初衷（不需要研究爆款選題）

---

### 項目 4：輸入欄位優化

#### 問題
- 目前的輸入欄位太抽象（「請輸入素材」）
- 用戶輸入不夠具體，AI 被迫編造

#### 做法

**設計原則**：
1. 引導問題是「可選」的，不是「強制」的
2. 只在用戶需要時展開
3. 不讓用戶覺得「我的素材不夠好」

**前端介面**：
```
輸入你的素材：
┌─────────────────────────────────────────────┐
│                                             │
│                                             │
│                                             │
└─────────────────────────────────────────────┘

💡 需要靈感？點擊展開引導問題 ▼

（展開後）
┌─────────────────────────────────────────────┐
│ 這些問題可以幫助你回想細節（選填）：            │
│                                             │
│ • 這件事發生在什麼時候？                      │
│ • 主角是誰？（可以是你、學員、朋友）           │
│ • 有沒有具體的數字或對話？                    │
│ • 你想傳達什麼感受或觀點？                    │
└─────────────────────────────────────────────┘
```

#### 預期效果
- 有需要的用戶可以獲得引導
- 不增加其他用戶的負擔
- 引導出更具體的素材，降低 AI 編造的需要

## 四、P2 優化項目詳細說明

### 項目 5：AI 詞彙替換後處理

#### 問題
- 即使提示詞禁止，AI 還是可能產生 AI 詞彙
- 需要在生成後再做一次檢查和替換

#### 做法

**建立 AI 詞彙替換表**：

| AI 詞彙 | 替換為 |
|---------|--------|
| 此外 | 然後、還有、而且 |
| 值得注意的是 | 有一點是、重點是 |
| 至關重要 | 很重要、超重要 |
| 深入探討 | 聊聊、說說 |
| 不僅...還... | ...而且... |
| 總而言之 | 總之、所以 |
| 綜上所述 | 所以、總之 |

**技術實作**：
- 在 LLM 生成後，用正則表達式檢測 AI 詞彙
- 自動替換為口語化詞彙
- 可選：顯示「已自動優化 X 處」

#### 預期效果
- 進一步降低 AI 感 10-20%
- 確保即使 LLM 沒遵守禁止規則，也能被後處理修正

---

### 項目 6：CTA 結尾多樣化

#### 問題
- 目前結尾常常是「你們覺得呢？」「留言區聊聊」
- 產生公式化的感覺

#### 做法

**建立 CTA 類型庫**：

| 類型 | 範例 |
|------|------|
| **問句型** | 你有類似的經驗嗎？ |
| **陳述型** | 就這樣，跟你分享一下。 |
| **反思型** | 不知道你怎麼想。 |
| **邀請型** | 有問題可以問我。 |
| **無 CTA** | （直接結束，不加結尾） |

**技術實作**：
- 根據貼文類型，選擇適合的 CTA 類型
- 在同一類型中隨機選擇具體 CTA
- 閒聊型、金句型可以「無 CTA」

#### 預期效果
- 結尾更多樣化
- 避免「每篇都問你們覺得呢」的問題

## 五、實施時程

| 階段 | 內容 | 時間 |
|------|------|------|
| **第一階段** | P0：提示詞精簡 + 各類型專屬結構 | 1-2 天 |
| **第二階段** | P1：選題靈感功能 + 輸入欄位優化 | 2-3 天 |
| **第三階段** | P2：AI 詞彙替換 + CTA 多樣化 | 1 天 |
| **測試驗收** | A/B 測試 + 用戶反饋 | 1 天 |

**總計：約 5-7 天**

---

## 六、驗收標準

| 指標 | 目標 |
|------|------|
| AI 感分數 | 從 15 分降到 5 分以下 |
| 用戶操作步驟 | 不增加（維持或減少） |
| 生成內容口語化程度 | 明顯提升 |
| 結尾多樣性 | 不再千篇一律「你們覺得呢」 |

---

## 七、風險與應對

| 風險 | 應對 |
|------|------|
| 精簡提示詞後內容品質下降 | 保留核心結構指引，只移除冗餘規則 |
| 選題推薦不準確 | 結合 IP 地基過濾，確保相關性 |
| 用戶不習慣新流程 | 保留原有入口，新功能為「可選」 |

---

## 八、總結

這次優化的核心目標是：

> **讓生成的內容更口語、更自然、更有人味，同時不增加用戶的操作負擔。**

關鍵改變：
1. **提示詞精簡**：從 7,500 字精簡到 1,000 字
2. **各類型專屬結構**：不再強制所有貼文都是「Hook + 故事 + CTA」
3. **選題靈感簡化**：給「1 個推薦」而不是「5 個選擇」
4. **引導問題可選**：不強制，只在需要時展開
5. **後處理優化**：AI 詞彙替換 + CTA 多樣化
