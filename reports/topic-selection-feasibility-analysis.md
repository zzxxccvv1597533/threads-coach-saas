## 二、三種選題情境的技術可行性

### 情境 1：完全沒有想法

**用戶需求**：「我不知道今天要寫什麼」

**技術方案**：

```
步驟 1：讀取用戶 IP 地基
  ↓ 取得 occupation（職業）、contentMatrixThemes（三大主題）
步驟 2：從 keywordBenchmarks 篩選相關關鍵字
  ↓ 根據用戶領域過濾
步驟 3：從 topicTemplates 推薦選題模板
  ↓ 根據關鍵字匹配
步驟 4：從 viralExamples 取得範例
  ↓ 顯示「這個選題的爆文範例」
步驟 5：AI 生成 3-5 個具體選題
  ↓ 結合用戶 IP 地基 + 爆款趨勢
```

**可行性**：✅ **完全可行**

- 所有需要的資料表都已存在
- 只需要新增一個 API 整合這些資源

---

### 情境 2：有點想法，想找切角

**用戶需求**：「我想寫關於『漲價』的內容，但不知道怎麼切入」

**技術方案**：

```
步驟 1：用戶輸入想法（如「漲價」）
  ↓
步驟 2：向量搜尋 viralExampleEmbeddings
  ↓ 找出語意相近的爆款貼文
步驟 3：分析這些爆款的「切角」
  ↓ 從 viralExamples 取得 contentType、funnelStage、opener50
步驟 4：用 AI 歸納 3-5 個不同切角
  ↓ 例如：故事型「我也曾經...」、知識型「3 個判斷方法」
步驟 5：每個切角附上爆文率和開頭範例
  ↓ 從 keywordBenchmarks 取得數據
```

**可行性**：✅ **完全可行**

- `viralExampleEmbeddings` 已有向量資料
- 只需要新增「向量搜尋 + 切角分析」的 API

---

### 情境 3：選題評估（爆紅機率判斷）

**用戶需求**：「我想寫這個主題，但不確定會不會爆」

**技術方案**：

```
步驟 1：用戶輸入選題（如「塔羅牌初學者指南」）
  ↓
步驟 2：向量搜尋 viralExampleEmbeddings
  ↓ 找出語意相近的爆款貼文
步驟 3：計算「爆紅機率」
  ↓ 基於：
     - 相似爆款的數量
     - 相似爆款的平均讚數
     - 關鍵字的 viralRate（從 keywordBenchmarks）
     - 用戶過去類似內容的表現（從 postMetrics）
步驟 4：給出評估結果
  ↓ 例如：「這個選題的爆紅機率約 35%，建議改用『反差型』切角」
步驟 5：如果機率低，推薦替代選題
  ↓ 從 topicTemplates 推薦相關但更有潛力的選題
```

**可行性**：✅ **完全可行**

- 所有需要的資料都已存在
- 需要設計「爆紅機率計算公式」

## 三、爆紅機率計算公式（建議）

```typescript
function calculateViralProbability(input: {
  similarViralCount: number;      // 相似爆款數量
  avgLikesOfSimilar: number;      // 相似爆款平均讚數
  keywordViralRate: number;       // 關鍵字爆文率（0-100）
  userHistoryViralRate: number;   // 用戶歷史爆文率（0-100）
  contentTypeMatch: boolean;      // 是否匹配用戶最佳內容類型
}): number {
  // 基礎分數（0-100）
  let score = 0;
  
  // 相似爆款數量（最多 30 分）
  score += Math.min(input.similarViralCount * 3, 30);
  
  // 相似爆款平均讚數（最多 20 分）
  if (input.avgLikesOfSimilar >= 1000) score += 20;
  else if (input.avgLikesOfSimilar >= 500) score += 15;
  else if (input.avgLikesOfSimilar >= 200) score += 10;
  else score += 5;
  
  // 關鍵字爆文率（最多 25 分）
  score += input.keywordViralRate * 0.25;
  
  // 用戶歷史表現（最多 15 分）
  score += input.userHistoryViralRate * 0.15;
  
  // 內容類型匹配加成（最多 10 分）
  if (input.contentTypeMatch) score += 10;
  
  return Math.min(score, 100);
}
```

**評估結果呈現**：

| 分數 | 評估 | 建議 |
|------|------|------|
| 70-100 | 🟢 高潛力 | 「這個選題很有潛力！建議直接開始創作」 |
| 40-69 | 🟡 中等 | 「這個選題有潛力，建議調整切角」 |
| 0-39 | 🔴 低潛力 | 「這個選題爆紅機率較低，建議換一個選題」 |

## 四、找不到相關爆款時的處理

**問題**：如果用戶輸入的主題在資料庫中找不到相關內容怎麼辦？

**解決方案**：

### 方案 A：誠實告知 + 通用建議

```
「這個主題在我們的爆款資料庫中沒有太多參考案例。

建議：
1. 加入具體的故事或數字，讓內容更有說服力
2. 使用『反差型』開頭，增加吸引力
3. 結尾加入互動問題，提升留言數」
```

### 方案 B：擴大搜尋範圍

```
步驟 1：降低向量相似度閾值
步驟 2：搜尋「相關但不完全相同」的爆款
步驟 3：提示用戶：「沒有完全相同的爆款，但這些相關內容可能有參考價值」
```

### 方案 C：使用通用爆款公式

```
步驟 1：從 viralClusters 取得「通用爆款公式」
步驟 2：推薦用戶使用這些公式
步驟 3：例如：「雖然沒有相關爆款，但根據我們的分析，『反差型 + 故事』的組合通常表現不錯」
```

**建議**：三種方案結合使用，優先 B → C → A

## 六、與 IP 地基和戰報的整合

### IP 地基整合

| 欄位 | 用途 |
|------|------|
| `occupation` | 過濾選題（只顯示相關領域） |
| `contentMatrixThemes` | 推薦選題的優先順序 |
| `personaEmotion` | 在引導問題中提示情感連結 |
| `bestPerformingType` | 推薦最適合用戶的貼文類型 |

### 戰報整合

| 欄位 | 用途 |
|------|------|
| `isViral` | 學習用戶的爆文特徵 |
| `viralAnalysis` | 提取成功因素 |
| `performanceLevel` | 調整推薦權重 |
| `contentType`（從 draftPosts） | 分析用戶最擅長的類型 |

## 八、結論

**現有資源完全可以支持三種選題情境**。

系統已經擁有：
- 1,739+ 篇爆款範例
- 向量 Embedding 資料
