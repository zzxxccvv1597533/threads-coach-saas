## 一、核心問題診斷

### 1.1 AI 感太重

經過分析，系統產生 AI 感的根本原因是：

| 問題 | 說明 |
|------|------|
| **提示詞過長** | 約 4,000-7,500 字，LLM 無法全部遵守，選擇性執行產生拼湊感 |
| **規則互相矛盾** | 「口語化」vs「結構化」同時要求 |
| **公式化結構** | 所有貼文都被強制套用「Hook + 故事 + CTA」 |
| **AI 詞彙殘留** | 即使禁止，LLM 仍可能產生「此外」「值得注意的是」等詞彙 |
| **結尾千篇一律** | 每篇都是「你們覺得呢？」「留言區聊聊」 |

### 1.2 選題概念混淆

之前的設計混淆了「選題」「Hook」「文案」三個層級：

| 層級 | 定義 | 長度 | 範例 |
|------|------|------|------|
| **選題** | 一個具體情境，代表這篇貼文要講什麼 | 15-40 字 | 「狗狗走了三個月了，他可能會想跟我說什麼嗎？」 |
| **Hook** | 貼文的開頭第一句話，吸引人繼續看 | 20-50 字 | 「上週有個客戶哭著來找我，說她的狗狗走了三個月...」 |
| **文案** | 完整的貼文內容 | 100-500 字 | 整篇貼文 |

**正確的選題定義**：

> 選題 = 一個具體情境描述（15-40 字），讓人想知道「然後呢？」

選題的特徵：
- 有具體場景（「在全家買茶葉蛋」「去咖啡廳」「拜月老」）
- 有事件或狀態（發生了什麼事，或處於什麼狀態）
- 有情緒張力（讓人想知道後續發展）
- 形式不限（可以是疑問句、陳述句、感嘆句）

## 二、系統架構重構

### 2.1 兩種創作模式

根據討論，系統將支援兩種創作模式：

| 模式 | 入口 | 流程 |
|------|------|------|
| **有想法** | 直接進入「發文工作室」 | 用戶有 Idea → 系統問幾個問題 → 生成文案 |
| **沒想法** | 先到「靈感工作室」 | 系統推薦選題（Tinder 式）→ 用戶選一個 → 進入發文工作室 |

### 2.2 靈感工作室（新功能）

**入口**：當用戶沒靈感、沒想法時點擊進入

**選題來源**：
1. **痛點矩陣**：根據用戶設定的受眾痛點，延伸出具體情境
2. **IP 資料推薦**：根據用戶的專業領域、人設、過往內容，推薦適合的選題
3. **爆款資料庫**：參考類似領域的爆款選題模式

**呈現方式**：Tinder 式，一次顯示一個選題

```
┌─────────────────────────────────────────────┐
│                                             │
│     「狗狗走了三個月了，                      │
│       他可能會想跟我說什麼嗎？」              │
│                                             │
│     💡 這個選題很適合你的風格                 │
│                                             │
│         [👎 換一個]  [👍 用這個]              │
│                                             │
└─────────────────────────────────────────────┘
```

**選題不重複機制**：
- 記錄用戶已經看過/使用過的選題
- 每次生成時確保不重複
- 提供新鮮的靈感

### 2.3 發文工作室（重構）

**流程**：

```
用戶輸入 Idea
    ↓
系統問幾個問題（引導更具體的素材）
    ↓
生成文案
    ↓
用戶可以對話式修改
```

**問幾個問題的設計**：
- 問題是「引導」而非「強制」
- 根據用戶輸入的 Idea 動態調整問題
- 目的是引導出更具體的素材，降低 AI 編造的需要

## 三、優化項目清單

### 3.1 P0 優先級（必須做）

| 項目 | 目標 | 預估時間 |
|------|------|----------|
| **提示詞精簡** | 從 4,000-7,500 字精簡到 1,000-1,500 字，降低 AI 感 40-50% | 4-6 小時 |
| **各貼文類型專屬結構** | 不再強制所有貼文都是「Hook + 故事 + CTA」 | 3-4 小時 |
| **選題定義修正** | 痛點矩陣產出「具體情境」而非「Hook」 | 2-3 小時 |

#### 提示詞精簡詳細說明

**修改前（約 4,000-7,500 字）**：
- 30+ 條規則
- 四透鏡框架
- 爆款元素
- 禁止事項
- 範例注入
- 互相矛盾的要求

**修改後（約 1,000-1,500 字）**：
```
你是一位 Threads 創作者，幫用戶把想法寫成貼文。

【核心原則】
1. 像跟朋友聊天一樣說話
2. 用真實的故事和感受
3. 句子要有長有短，有節奏

【禁止詞彙】
此外、值得注意的是、至關重要、深入探討、不僅...還...

【口語詞彙（鼓勵使用）】
欸、啊、真的、超、蠻、其實、然後、就是

【結構建議（不強制）】
- 開頭：直接說重點或問問題
- 中間：說故事或分享觀點
- 結尾：可以問問題，也可以不問
```

#### 各貼文類型專屬結構

| 類型 | 結構 | 結尾 |
|------|------|------|
| **故事型** | 時間 → 衝突 → 轉折 → 啟發 | 反思（不一定要問問題） |
| **知識型** | 問題 → 重點 → 行動建議 | 行動建議 |
| **閒聊型** | 情緒 → 碎片 → 隨意 | **可以沒有結尾** |
| **提問型** | 問題 → 背景 | 問題本身 |
| **觀點型** | 觀點 → 論點 → 討論 | 討論或反問 |
| **反差型** | 預期 → 反轉 → 解釋 | 可有可無 |
| **對話型** | 問題 → 回答 → 延伸 | 延伸思考 |
| **診斷型** | 症狀 → 分析 → 建議 | 建議或鼓勵 |
| **清單型** | 主題 → 列點 → 總結 | 總結或邀請 |
| **金句型** | 金句 → 解釋 → 共鳴 | 共鳴或留白 |

### 3.2 P1 優先級（重要）

| 項目 | 目標 | 預估時間 |
|------|------|----------|
| **靈感工作室** | 解決「不知道寫什麼」的問題 | 8-10 小時 |
| **發文工作室問答流程** | 用戶輸入 Idea 後，系統問幾個問題再生成 | 6-8 小時 |
| **選題不重複機制** | 每次生成的選題不與之前重複 | 2-3 小時 |

#### 靈感工作室詳細說明

**功能**：
1. 從痛點矩陣 + IP 資料 + 爆款資料庫生成選題
2. Tinder 式呈現，一次一個
3. 用戶可以「換一個」或「用這個」
4. 選擇後進入發文工作室

**選題生成邏輯**：
```
1. 取得用戶的 IP 資料（領域、風格、受眾）
2. 取得用戶設定的痛點矩陣
3. 從爆款資料庫找相關領域的選題模式
4. 結合以上三者，生成「具體情境」的選題
5. 過濾掉用戶已經看過/使用過的選題
6. 輸出一個選題
```

#### 發文工作室問答流程

**設計原則**：
- 問題是「引導」而非「強制」
- 問題數量 2-3 個，不要太多
- 根據用戶輸入的 Idea 動態調整

**範例流程**：
```
用戶：「我想寫關於漲價的焦慮」

AI：「好的！讓我問你幾個問題，幫助我更了解你想說什麼：

1. 這是你自己的經歷，還是學員/客戶的故事？
2. 有沒有具體的數字？（例如：從多少漲到多少）
3. 最後的結果是什麼？（成功漲價？還是還在猶豫？）」

用戶回答後 → 生成文案
```

### 3.3 P2 優先級（加分）

| 項目 | 目標 | 預估時間 |
|------|------|----------|
| **AI 詞彙替換後處理** | 進一步降低 AI 感 10-20% | 2-3 小時 |
| **CTA 結尾多樣化** | 避免「每篇都問你們覺得呢」 | 2-3 小時 |
| **風格 DNA 提取** | 從「複製句式」改為「學習風格」 | 4-6 小時 |

#### AI 詞彙替換表

| AI 詞彙 | 替換為 |
|---------|--------|
| 此外 | 然後、還有、而且 |
| 值得注意的是 | 有一點是、重點是 |
| 至關重要 | 很重要、超重要 |
| 深入探討 | 聊聊、說說 |
| 不僅...還... | ...而且... |
| 總而言之 | 總之、所以 |
| 綜上所述 | 所以、總之 |

#### CTA 結尾類型庫

| 類型 | 範例 | 適用貼文類型 |
|------|------|--------------|
| **問句型** | 你有類似的經驗嗎？ | 故事型、觀點型 |
| **陳述型** | 就這樣，跟你分享一下。 | 閒聊型、故事型 |
| **反思型** | 不知道你怎麼想。 | 觀點型、反差型 |
| **邀請型** | 有問題可以問我。 | 知識型、診斷型 |
| **無 CTA** | （直接結束，不加結尾） | 閒聊型、金句型 |
