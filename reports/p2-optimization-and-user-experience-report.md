# P2 優化項目詳細說明 + 用戶感受差異分析

生成日期：2025-01-20

## 一、P2 優化項目詳細說明

### 1. 內容靈感發想功能（4-6 小時）

#### 功能定位
解決學員「不知道今天要發什麼」的核心痛點，利用 603 個成功因素 + 2,542 個爆款 Embedding，為學員提供個人化的選題建議。

#### 功能設計

**入口位置**：Dashboard 首頁 + 發文工作室「沒靈感」區域

**核心流程**：
1. 系統讀取學員的 IP 地基（專業領域、受眾、人設三支柱）
2. 根據學員領域，匹配相關的成功因素
3. 結合當前經營階段，推薦適合的主題方向
4. 每次給出 5-10 個具體可發的選題

**輸出範例**：
```
📌 今日推薦選題（根據你的 IP 地基分析）

1. 【故事型】分享你第一次遇到 XX 困難的經歷
   → 爆款率 28%，適合「讓人更懂我」目標
   
2. 【觀點型】為什麼我認為 XX 比 YY 更重要
   → 爆款率 22%，適合「讓人信任我」目標
   
3. 【提問型】你們覺得 XX 還是 YY？
   → 爆款率 35%，適合「有人互動留言」目標
```

**技術實現**：
- 整合 `getSpiritualSuccessFactors()` 取得成功因素
- 根據學員領域篩選相關因素
- 結合經營階段權重排序
- 使用 LLM 生成具體選題

#### 預期效果
- 解決「不知道發什麼」痛點
- 選題成功率提升 30-50%
- 減少學員思考時間

### 2. 內容組合策略（4-6 小時）

#### 功能定位
從「寫單篇貼文」升級到「經營帳號」，幫助學員建立內容節奏，知道「這週該發什麼類型的內容」。

#### 功能設計

**入口位置**：Dashboard 首頁「本週建議」區塊

**核心邏輯**：
1. 分析 50 個成功 IP 的內容組合比例
2. 追蹤學員過去 7 天的發文類型
3. 計算學員目前的內容比例
4. 建議下一篇該發什麼類型

**內容組合黃金比例**（基於 50 個 IP 分析）：
| 內容類型 | 建議比例 | 說明 |
|----------|----------|------|
| 價值輸出（知識/故事/觀點） | 50% | 建立專業形象 |
| 互動型（提問/投票/對話） | 30% | 提高互動率 |
| 日常閒聊 | 15% | 展現真實人設 |
| 變現導向 | 5% | 自然帶入產品 |

**輸出範例**：
```
📊 你的內容組合分析（過去 7 天）

價值輸出：2 篇（40%）→ 建議再發 1 篇
互動型：0 篇（0%）→ ⚠️ 缺少互動內容
日常閒聊：3 篇（60%）→ 比例偏高

💡 建議：今天發一篇「提問型」或「投票型」貼文
   推薦主題：「你們覺得 XX 還是 YY？」
```

**技術實現**：
- 追蹤 `draft_posts` 表的 `contentType` 欄位
- 計算過去 7 天各類型比例
- 與黃金比例對比，找出缺口
- 推薦下一篇類型

#### 預期效果
- 帳號內容更均衡
- 避免「只發同一種類型」的問題
- 建立穩定的內容節奏

### 3. 帳號健康度診斷（4-6 小時）

#### 功能定位
讓學員了解自己帳號的「健康狀態」，提供數據驅動的改善建議。

#### 功能設計

**入口位置**：側邊欄「戰報分析」→「帳號診斷」

**診斷維度**：
1. **內容健康度**：發文頻率、類型分布、字數分布
2. **互動健康度**：平均留言數、留言回覆率
3. **成長健康度**：粉絲成長趨勢、觸及變化
4. **人設一致性**：內容是否符合 IP 地基設定

**輸出範例**：
```
🏥 帳號健康度診斷報告

總體評分：72/100 分

📝 內容健康度：80/100
   ✅ 發文頻率穩定（每週 5 篇）
   ⚠️ 類型分布不均（閒聊型佔 60%）
   
💬 互動健康度：65/100
   ⚠️ 平均留言數偏低（3.2 則/篇）
   ❌ 留言回覆率過低（20%）
   
📈 成長健康度：70/100
   ✅ 粉絲穩定成長中
   ⚠️ 觸及率有下降趨勢

🎯 改善建議：
1. 增加互動型內容（提問/投票）提高留言數
2. 每篇貼文至少回覆 3 則留言
3. 嘗試「反差型」開頭提高觸及
```

**技術實現**：
- 整合 `post_metrics` 表的數據
- 計算各維度分數
- 與 50 個 IP 的平均值對比
- 生成個人化建議

#### 預期效果
- 學員清楚知道問題在哪
- 提供具體可行的改善方向
- 增加產品黏性

### 4. 各類型填空差異化（1 小時）

#### 功能定位
強化不同貼文類型的輸入引導，幫助學員提供更完整的素材。

#### 現狀分析
目前已有基本的類型差異化，但部分類型的引導不夠具體。

#### 優化方向

**故事型**：
- 現有：事件/衝突、轉折點、情感變化、核心啟發
- 優化：增加「故事發生的時間地點」「當時的心情」

**觀點型**：
- 現有：觀察到的現象、你的立場、背後價值觀
- 優化：增加「你為什麼這樣想」「反對意見是什麼」

**診斷型**：
- 現有：特徵/症狀、診斷標籤、解析
- 優化：增加「這類人的典型行為」「解決方案暗示」

#### 預期效果
- 學員更容易填寫素材
- 生成內容更完整
- 減少「素材不足」導致的低質量輸出

## 二、P0+P1 優化後的用戶感受差異

### 優化前 vs 優化後對比

#### 1. 腦力激盪（brainstorm）

| 維度 | 優化前 | 優化後 |
|------|--------|--------|
| 主題來源 | 純 LLM 隨機發想 | 參考 2,542 篇爆款的語意匹配 |
| 目標影響 | 選「讓人更懂我」或「慢慢賣產品」結果一樣 | 不同目標產生明顯不同的主題建議 |
| 用戶感受 | 「建議太泛，不知道怎麼用」 | 「建議很精準，直接可以發」 |

**具體差異範例**：

優化前（選擇「讓人更懂我」）：
```
1. 分享你的專業知識
2. 寫一篇觀點文
3. 發一個提問
```

優化後（選擇「讓人更懂我」）：
```
1. 【故事型】分享你第一次接觸身心靈的契機
   → 參考爆款：「三年前的我，根本不相信...」（5,200 讚）
   
2. 【觀點型】為什麼我選擇成為療癒師而不是上班族
   → 參考爆款：「很多人問我為什麼放棄高薪...」（3,800 讚）
   
3. 【閒聊型】今天的一個小感悟
   → 參考爆款：「剛剛散步的時候突然想到...」（2,100 讚）
```

#### 2. 切角分析（analyzeAngles）

| 維度 | 優化前 | 優化後 |
|------|--------|--------|
| 切角來源 | 純 LLM 創意 | 參考相似爆款的切角方式 |
| 開頭示範 | 固定模式 | 參考真實爆款開頭 |
| 用戶感受 | 「切角太普通」 | 「切角有數據支撐，更有信心」 |

**具體差異範例**：

素材：「今天有個學員跟我說，她終於敢表達自己了」

優化前：
```
切角 1：故事型 - 用學員的故事帶出觀點
切角 2：觀點型 - 直接表達你對「表達自己」的看法
切角 3：提問型 - 問大家有沒有類似經驗
```

優化後：
```
切角 1：故事型 [相似度 85%]
   參考爆款：「她說這句話的時候，眼眶紅了...」（4,500 讚）
   開頭示範：「今天收到一則訊息，看完我愣了三秒」
   
切角 2：反差型 [相似度 78%]
   參考爆款：「以前的她，連點餐都不敢...」（3,200 讚）
   開頭示範：「三個月前，她連在群組打字都會刪掉重打」
   
切角 3：對話型 [相似度 72%]
   參考爆款：「『老師，我做到了』她說...」（2,800 讚）
   開頭示範：「『我今天跟老闆說了我的想法』她傳訊息給我」
```

#### 3. 開頭生成（generateHooks）

| 維度 | 優化前 | 優化後 |
|------|--------|--------|
| 開頭來源 | 13 種固定模式 | 語意匹配真實爆款開頭 |
| 多樣性 | 容易重複 | 每次參考不同爆款，更有變化 |
| 用戶感受 | 「開頭都差不多」 | 「開頭更自然、更有吸引力」 |

**具體差異範例**：

主題：「為什麼我不建議新手一開始就學太多技術」

優化前：
```
Hook 1：你是不是也常常覺得學不完？
Hook 2：很多人以為學越多越好，但其實...
Hook 3：昨天有人問我，新手該學什麼？
```

優化後：
```
Hook 1：[鏡像式] 「學了三個月，發現自己什麼都不會」
   → 參考爆款（4,200 讚）：直接說出新手的心聲
   
Hook 2：[反差式] 「我花了兩年才明白，少即是多」
   → 參考爆款（3,800 讚）：用個人經歷製造反差
   
Hook 3：[數據式] 「90% 的新手都會犯這個錯」
   → 參考爆款（5,100 讚）：用數字抓注意力
   
Hook 4：[場景式] 「上週有個學員跟我說，她買了 20 本書」
   → 參考爆款（2,900 讚）：用具體場景帶入
   
Hook 5：[對話式] 「『老師，我該先學什麼？』這是我最常被問的問題」
   → 參考爆款（3,500 讚）：用真實對話開場
```

#### 4. 內容生成（generateDraft）

| 維度 | 優化前 | 優化後 |
|------|--------|--------|
| 選定開頭 | 可能被 AI 改掉 | 100% 保留用戶選定的開頭 |
| 受眾差異 | 不同受眾內容差不多 | 明顯的語言風格差異 |
| 類型差異 | 故事型/觀點型結構相似 | 每種類型有專屬結構 |
| 虛構場景 | 常出現「我媽突然問我」 | 禁止虛構對話 |
| 成功因素 | 未整合 | 603 個成功因素自動參考 |

**受眾差異範例**：

同一主題「如何開始冥想」，不同受眾的輸出差異：

**受眾：完全新手**
```
「冥想」這兩個字，聽起來好像很難對不對？

其實不用盤腿、不用點香、不用唸咒語。

你只需要：
找一個安靜的地方坐下來
閉上眼睛
專注在呼吸上

就這樣，這就是冥想。

一開始 30 秒就好，不用逼自己。
慢慢來，你會發現...
```

**受眾：有經驗的修行者**
```
練習冥想三年後，我發現一個關鍵：

不是「專注」，而是「覺察」。

很多人把冥想當成「控制念頭」的練習，
但真正的冥想是「觀察念頭來去」。

這個轉變讓我的練習從「努力」變成「放鬆」。

你有過類似的體悟嗎？
```

#### 5. 文章類型差異

**故事型**（優化後必須具備）：
- ✅ 有具體的時間地點
- ✅ 有情緒轉折點
- ✅ 有個人感悟
- ✅ 有「後來...」的發展

**對話型**（優化後必須具備）：
- ✅ 有明確的對話者
- ✅ 有「」標註的對話內容
- ✅ 有對話後的反思
- ❌ 禁止「我媽突然問我」等虛構場景

**觀點型**（優化後必須具備）：
- ✅ 有明確的立場（「我認為...」）
- ✅ 有 2-3 個支撐論點
- ✅ 有邀請討論的結尾
- ❌ 不能只是描述現象

## 三、用戶體驗改善總結

### 量化指標預估

| 指標 | 優化前 | 優化後 | 提升幅度 |
|------|--------|--------|----------|
| 主題建議精準度 | 50% | 80% | +60% |
| 切角建議實用性 | 60% | 85% | +42% |
| 開頭吸引力 | 65% | 90% | +38% |
| 開頭一致性 | 70% | 100% | +43% |
| 受眾匹配度 | 55% | 85% | +55% |
| 類型差異化 | 40% | 90% | +125% |

### 用戶感受變化

**優化前的典型抱怨**：
- 「系統給的建議太泛，不知道怎麼用」
- 「選了開頭，結果生成的不一樣」
- 「選不同受眾，內容差不多」
- 「故事型和觀點型看起來一樣」
- 「為什麼 AI 老是寫『我媽突然問我』」

**優化後的預期反饋**：
- 「建議很精準，直接可以發」
- 「選的開頭完整保留了」
- 「針對新手寫的內容真的比較簡單」
- 「故事型有起承轉合，觀點型有論點論據」
- 「內容更真實，沒有虛構的對話了」
