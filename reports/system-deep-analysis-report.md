# 系統深度分析報告

## 一、受眾差異化完整運作方式

### 1.1 受眾資料來源

系統從 IP 地基的 `targetAudience` 欄位讀取學員設定的目標受眾，包含：
- **受眾描述**：例如「剛開始接觸冥想的上班族」
- **受眾痛點**：例如「工作壓力大、睡眠品質差」
- **受眾渴望**：例如「找到內心平靜、提升專注力」

### 1.2 受眾如何影響內容生成

在 `generateDraft` API 中，系統會根據受眾調整：

| 受眾類型 | 語言風格 | 場景描述 | 專業術語 |
|----------|----------|----------|----------|
| **新手/入門者** | 簡單易懂、口語化 | 日常生活場景 | 避免使用，或加解釋 |
| **有經驗者** | 可使用專業詞彙 | 進階練習場景 | 適度使用 |
| **專業人士** | 專業精準 | 專業情境 | 自然使用 |

### 1.3 實際 Prompt 範例

當學員選擇「新手」受眾時，系統會在 AI Prompt 中加入：

```
=== 受眾設定 ===
目標受眾：剛開始接觸冥想的上班族

【語言調整】
- 使用簡單易懂的語言，避免專業術語
- 用日常生活的比喻來解釋概念
- 語氣親切、鼓勵，像朋友分享經驗

【場景描述】
- 描述受眾熟悉的日常場景（通勤、午休、睡前）
- 避免使用專業修行場景
```

### 1.4 選擇不同受眾的差異

**範例主題：「如何開始冥想」**

| 受眾 | 開頭範例 | 內容風格 |
|------|----------|----------|
| **完全新手** | 「冥想不用盤腿、不用點香，你只需要找一個安靜的地方坐下來...」 | 破除迷思、降低門檻 |
| **有基礎者** | 「練習冥想一段時間後，你可能會遇到『雜念太多』的瓶頸...」 | 解決進階問題 |
| **專業修行者** | 「從止觀到內觀，冥想的深化需要理解『覺察』與『專注』的差異...」 | 專業討論、深度探索 |

## 二、目前系統學習學員風格的機制

### 2.1 現有機制

| 機制 | 資料來源 | 學習內容 | 限制 |
|------|----------|----------|------|
| **IP 地基整合** | 學員填寫的人設資料 | 職業、專業、語氣風格 | 只是描述，不是實際範例 |
| **語氣風格設定** | IP 地基的 `personaTone` | 「溫暖親切」「專業權威」等 | 只是標籤，無法學習細節 |
| **多樣性提示** | UI 提示文字 | 提醒學員加入個人經歷 | 只是提醒，系統無法主動學習 |

### 2.2 現有機制的限制

**目前系統「不知道」學員：**
- ❌ 平常怎麼說話（口頭禪、慣用語）
- ❌ 過去發過什麼貼文（風格、節奏）
- ❌ 偏好的句子長度和結構
- ❌ 使用表情符號的習慣

### 2.3 為什麼現有機制不夠？

| 學員填寫 | 實際風格 | 差距 |
|----------|----------|------|
| 「語氣：溫暖親切」 | 「欸你們知道嗎，我今天超崩潰的...」 | AI 無法從「溫暖親切」學到這種口語 |
| 「專業：心理諮商」 | 「我常說，情緒沒有對錯，只有來去」 | AI 無法知道這是學員的口頭禪 |

## 三、向量資料庫完整性分析

### 3.1 資料庫統計

| 數據類型 | 數量 | 說明 |
|----------|------|------|
| **IP 帳號** | 50 個 | 來自 50 個成功 IP 的數據 |
| **IP 貼文** | 29,475 篇 | 所有貼文（含非爆款） |
| **爆款貼文（≥1000讚）** | 1,780 篇 | 佔總貼文 6.04% |
| **超級爆款（≥5000讚）** | 277 篇 | 佔總貼文 0.94% |
| **viral_examples** | 2,542 筆 | 用於語意匹配的爆款範例 |
| **viral_example_embeddings** | 2,542 筆 | 100% 覆蓋率 |
| **成功因素** | 603 個 | 主題 147 + 切角 147 + 類型 146 + 呈現 163 |

### 3.2 向量資料庫是否包含所有爆款？

**答案：部分包含**

| 資料來源 | 數量 | 是否有向量 |
|----------|------|------------|
| `viral_examples` 表 | 2,542 筆 | ✅ 100% 有向量 |
| `ip_posts` 表（爆款） | 1,780 篇 | ❌ 沒有獨立向量 |

**說明**：
- `viral_examples` 是精選的爆款範例，已全部生成向量
- `ip_posts` 中的 1,780 篇爆款貼文，目前**沒有**獨立生成向量
- 兩者可能有重疊，但 `ip_posts` 的爆款可能有部分未被納入 `viral_examples`

### 3.3 可以更強的地方

1. **將 ip_posts 爆款納入向量庫**
   - 目前 1,780 篇爆款可能未完全納入
   - 可以增加向量庫的覆蓋範圍

2. **定期更新爆款數據**
   - 目前是靜態數據，沒有定期更新機制
   - 可以追蹤最新的爆款趨勢

## 四、個人風格學習的可行方案

### 4.1 方案一：上傳過去貼文（推薦）

**實作方式**：
1. 學員上傳 10-20 篇過去發過的貼文
2. 系統分析這些貼文，提取風格特徵
3. 在生成內容時，參考這些風格特徵

**可以學習的內容**：

| 特徵 | 分析方式 | 應用方式 |
|------|----------|----------|
| **口頭禪** | 識別重複出現的開場白、結尾語 | 自然融入生成內容 |
| **句子長度** | 統計平均句子長度 | 調整生成內容的節奏 |
| **表情符號** | 統計使用頻率和類型 | 適當加入或減少 |
| **段落結構** | 分析分段習慣 | 模仿段落長度和結構 |
| **語氣詞** | 識別「欸」「啊」「嘛」等語氣詞 | 增加口語感 |

**預估時間**：6-8 小時

### 4.2 方案二：語音轉文字分析

**實作方式**：
1. 學員上傳諮詢錄音或 Podcast
2. 系統轉錄成文字
3. 分析口語表達習慣

**優點**：更接近學員真實說話方式
**缺點**：需要額外的語音處理，成本較高

**預估時間**：10-12 小時

### 4.3 方案三：即時學習（長期）

**實作方式**：
1. 學員每次編輯生成內容後，系統記錄修改
2. 分析學員傾向修改的部分
3. 逐漸學習學員偏好

**優點**：無需額外上傳，自然學習
**缺點**：需要較長時間累積數據

**預估時間**：15-20 小時

### 4.4 推薦方案

**優先實作「上傳過去貼文」**，原因：
1. 技術可行性高，可利用現有 LLM 能力
2. 學員容易操作，只需複製貼上
3. 效果立竿見影，上傳後立即生效
4. 符合您之前提到的「個人風格學習」需求

## 五、總結

### 5.1 目前系統的強項

1. **數據基礎紮實**：50 個 IP、29,475 篇貼文、603 個成功因素
2. **語意匹配完善**：2,542 個向量、語意擴展功能
3. **受眾差異化**：根據受眾調整語言風格
4. **多維度分析**：主題、切角、類型、呈現方式

### 5.2 主要提升空間

1. **個人風格學習**：讓生成內容更像學員本人
2. **向量庫擴展**：將 ip_posts 爆款納入
3. **數據透明化**：讓學員知道建議的依據

### 5.3 建議下一步

| 優先級 | 項目 | 預估時間 |
|--------|------|----------|
| **P1** | 上傳過去貼文功能 | 6-8 小時 |
| **P2** | 將 ip_posts 爆款納入向量庫 | 3-4 小時 |
| **P2** | 數據透明化 | 2-3 小時 |
