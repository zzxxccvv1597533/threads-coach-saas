## 二、Embedding 技術架構

### 2.1 向量生成方式

目前系統使用**本地文字特徵提取**生成 256 維向量，**不依賴外部 API**。

```typescript
// 向量維度結構（256 維）
vector[0-63]   = 字符頻率分布
vector[64-127] = N-gram 哈希
vector[128-191] = 詞彙特徵（問句、數字、開頭模式等）
vector[192-255] = 詞彙哈希
```

### 2.2 詞彙特徵提取（關鍵）

系統會提取以下特徵：

| 特徵 | 維度 | 說明 |
|------|------|------|
| 標點符號 | 128 | 是否含 ！？。，等 |
| Emoji | 129 | 是否含表情符號 |
| 換行數 | 130 | 段落結構 |
| 詞數 | 131 | 文章長度 |
| 字數 | 132 | 精確長度 |
| 中文比例 | 133 | 中文字佔比 |
| 數字比例 | 134 | 數字佔比 |
| 問句 | 135 | 是否含問號 |
| 驚嘆 | 136 | 是否含驚嘆號 |
| 「我」開頭 | 137 | 第一人稱開頭 |
| 「你」開頭 | 138 | 第二人稱開頭 |
| 數字開頭 | 139 | 列表型開頭 |

### 2.3 相似度計算

使用 **Cosine Similarity** 計算兩個向量的相似度：

```typescript
similarity = dotProduct(vecA, vecB) / (magnitude(vecA) * magnitude(vecB))
// 返回值範圍：-1 到 1，1 表示完全相同
```

## 三、目前的數據利用方式

### 3.1 選題階段（brainstorm）

**觸發時機**：學員輸入主題後

**數據利用**：
```typescript
// 1. 用學員輸入的主題生成 Embedding
const inputEmbedding = await generateEmbedding(input.topic);

// 2. 與 2,542 個爆款向量比對
const similarExamples = await findSimilarViralExamples(input.topic, 5);

// 3. 返回相似度最高的 5 篇爆款
// 格式：[相似度 78%] 練習冥想 3 年，我發現...
```

**實際參考內容**：
- 相似爆款的**完整貼文內容**（前 80 字）
- 相似度百分比
- 讚數

**問題**：
- ❌ 只顯示「參考」，沒有給出「爆款潛力評分」
- ❌ 學員不知道這個主題好不好
- ❌ 沒有告訴學員「如何調整主題更容易爆」

---

### 3.2 切角分析階段（analyzeAngles）

**觸發時機**：學員輸入素材後

**數據利用**：
```typescript
// 1. 用學員的素材生成 Embedding
const inputEmbedding = await generateEmbedding(input.material);

// 2. 找出相似的爆款切角
const similarExamples = await findSimilarViralExamples(input.material, 3);
```

**實際參考內容**：
- 相似爆款的**切角和開頭方式**
- 相似度百分比

---

### 3.3 開頭生成階段（generateHooks）

**觸發時機**：學員選擇主題和類型後

**數據利用**：
```typescript
// 1. 用主題生成 Embedding
const inputEmbedding = await generateEmbedding(input.topic);

// 2. 找出相似主題的爆款開頭
const similarExamples = await findSimilarViralExamples(input.topic, 5);
```

**實際參考內容**：
- 相似爆款的**開頭前 50 字**
- 讚數
- 相似度百分比

---

### 3.4 全文生成階段（generateDraft）

**觸發時機**：學員點擊「生成全文」

**數據利用**：
```typescript
// 1. 取得成功因素
const factors = await getSpiritualSuccessFactors();

// 2. 取得內容長度建議
const contentRecs = await getContentTypeRecommendations();

// 3. 取得呈現方式建議
const presentationRecs = await getPresentationRecommendations();
```

**實際參考內容**：
- 高爆款率主題（如「自我覺察」爆款率 35%）
- 高爆款率切角（如「反差對比」爆款率 42%）
- 內容長度建議（100-200 字爆款率 62%）
- 呈現方式建議（含問句爆款率 49%）

## 四、Embedding 匹配的實際效果

### 4.1 匹配範例

假設學員輸入主題：「冥想入門」

系統會：
1. 將「冥想入門」轉換為 256 維向量
2. 與 2,542 個爆款向量逐一計算 Cosine Similarity
3. 返回相似度最高的 5 篇

**返回結果範例**：
```
1. [相似度 78%] 練習冥想 3 年，我發現最重要的不是技巧，而是...（讚 2,341）
2. [相似度 72%] 冥想不是放空，而是學會和自己的念頭共處...（讚 1,892）
3. [相似度 68%] 為什麼我每天只冥想 5 分鐘，卻比以前 30 分鐘更有效...（讚 1,567）
4. [相似度 65%] 很多人問我怎麼開始冥想，我的答案是...（讚 1,234）
5. [相似度 61%] 冥想改變了我的人生，但不是你想的那種改變...（讚 1,123）
```

### 4.2 匹配原理

為什麼「冥想入門」會匹配到「練習冥想 3 年」？

因為兩者在以下特徵上相似：
- **N-gram 哈希**：「冥想」這個詞的 3-gram 相同
- **詞彙哈希**：「冥想」這個詞的哈希值相同
- **中文比例**：都是高中文比例的內容

### 4.3 匹配的局限性

目前的 Embedding 是**基於文字特徵**，而非**語意理解**：

| 匹配類型 | 效果 |
|----------|------|
| 相同詞彙 | ✅ 很好 |
| 同義詞 | ⚠️ 一般 |
| 語意相似但詞彙不同 | ❌ 較差 |

例如：
- 「冥想入門」→「冥想練習」：✅ 會匹配（詞彙相似）
- 「冥想入門」→「靜心方法」：⚠️ 可能匹配（部分特徵相似）
- 「冥想入門」→「正念練習」：❌ 可能不匹配（詞彙不同）

## 五、您提到的「領讀時跟旭東老師聊過的內容」

### 5.1 目前系統能抓到什麼？

如果您在選題時輸入「領讀」或「旭東老師」相關的內容，系統會：

1. **Embedding 匹配**：找出與「領讀」相似的爆款
2. **關鍵字匹配**：從 52 個關鍵字中找出相關的市場數據
3. **成功因素匹配**：從 603 個成功因素中找出相關的建議

### 5.2 目前系統抓不到什麼？

- ❌ **對話脈絡**：系統不知道您和旭東老師聊了什麼
- ❌ **個人化知識**：系統沒有儲存您的個人筆記或對話記錄
- ❌ **領域專屬知識**：如果「領讀」這個詞在爆款庫中很少出現，匹配效果會較差

### 5.3 如何改善？

**方案 A：擴充知識庫**
- 將您和旭東老師的對話重點整理成「個人知識庫」
- 在選題時，系統同時查詢「爆款庫」和「個人知識庫」

**方案 B：強化 IP 地基連結**
- 在 IP 地基中增加「個人經歷」欄位
- 選題時，系統會優先匹配與您個人經歷相關的爆款

**方案 C：使用外部 Embedding API**
- 改用 OpenAI 或其他語意 Embedding API
- 可以理解同義詞和語意相似的內容
- 缺點：需要額外成本
