## 第一部分：Manus 內建 API 的 Embedding 支援分析

### 測試結果

| API Endpoint | 狀態 | 說明 |
|--------------|------|------|
| `/v1/embeddings` | ❌ 404 Not Found | Manus Forge API **不支援**獨立的 Embedding endpoint |
| `/v1/chat/completions` | ✅ 200 OK | LLM API 正常運作 |

### 結論

**您不需要提供外部 API Key**，但 Manus 內建 API 目前沒有獨立的 Embedding 功能。

### 替代方案：LLM 語意擴展

雖然沒有 Embedding API，但可以透過 **LLM 語意擴展** 達到類似效果：

```
輸入：「冥想」
LLM 輸出：["靜坐", "沉思", "禪修", "正念", "內觀"]
```

**優點**：
- 不需要外部 API Key
- 語意理解能力強
- 可以理解同義詞和相關概念

**缺點**：
- 每次查詢需要 LLM 調用（約 0.5-1 秒延遲）
- 無法預先計算向量，需要即時處理

## 第三部分：待優化項目

### 3.1 語意 Embedding 升級（核心優化）

**目前問題**：
- 現有 Embedding 基於字符特徵，非語意理解
- 同義詞匹配效果差（冥想 ↔ 靜心）
- 語意相似但詞彙不同時匹配失敗

**建議方案**：LLM 語意擴展 + 多詞彙匹配

```
Step 1: 學員輸入「靜心」
Step 2: LLM 擴展為 ["靜心", "冥想", "禪修", "正念", "內觀"]
Step 3: 用所有詞彙分別匹配向量庫
Step 4: 合併結果，去重後返回
```

**預估時間**：3-4 小時  
**風險**：增加約 0.5-1 秒延遲

---

### 3.2 相關性提示（輔助功能）

**目前問題**：
- 即使匹配結果不相關，系統仍顯示
- 學員不知道結果是否可靠

**建議方案**：當相似度低於 50% 時顯示提示

```
💡 目前爆款庫中沒有與「區塊鏈」高度相關的範例。
   以下僅供啟發參考，建議嘗試用更常見的詞彙描述你的主題。
```

**預估時間**：1-2 小時  
**風險**：無

---

### 3.3 開頭選擇優化（移除評分，改為建議）

**您的要求**：不要有分數評分，避免學員產生負面想法

**建議方案**：改為「參考提示」形式

```
⭐ 推薦開頭（與爆款風格相近）
   「練習冥想 3 年，我發現最重要的不是技巧...」
   → 這個開頭的風格接近爆款「練習冥想 3 年，我發現...」

其他選項
   「很多人問我怎麼開始冥想...」
```

**預估時間**：2-3 小時  
**風險**：無

---

### 3.4 類型智能推薦

**目前問題**：
- 學員不知道「這個主題用哪種類型最容易成功」

**建議方案**：根據相似爆款的類型分佈推薦

```
💡 根據相似爆款分析：
   - 故事型：12 篇相似爆款使用此類型
   - 知識型：8 篇相似爆款使用此類型
   
   建議：這個主題用「故事型」可能更適合
```

**預估時間**：2-3 小時  
**風險**：無

## 第六部分：風險評估

### 主要風險

| 風險 | 等級 | 說明 | 緩解措施 |
|------|------|------|----------|
| **內容同質化** | 🔴 高 | 所有學員都選「推薦」選項 | 加入多樣性提示、強調個人風格 |
| **過度依賴工具** | 🟡 中 | 學員忽略內容本質 | 保留「自由選擇」、強調「參考而非複製」 |
| **LLM 延遲** | 🟡 中 | 語意擴展增加 0.5-1 秒 | 可接受，用戶體驗影響小 |
| **數據偏差** | 🟡 中 | 新領域可能沒有足夠參考 | 顯示「數據不足」提示 |

### 設計原則（符合您的定位）

> **工具是輔助角色，不是取代學員創作**

1. **不使用評分**：改為「建議」和「參考」
2. **保留自由選擇**：所有選項都可選，不強制
3. **強調個人風格**：提示學員加入自己的經歷
4. **透明化數據來源**：讓學員知道建議的依據

## 第七部分：技術架構變更

### 語意 Embedding 升級架構

```
目前架構：
輸入 → 本地特徵提取 → 256 維向量 → Cosine Similarity → 結果

升級架構：
輸入 → LLM 語意擴展 → 多個關鍵詞 → 分別匹配向量庫 → 合併結果 → 結果
```

### 新增 API

| API | 說明 |
|-----|------|
| `expandSemanticKeywords` | 用 LLM 擴展同義詞 |
| `findSimilarWithSemanticExpansion` | 語意擴展 + 向量匹配 |
