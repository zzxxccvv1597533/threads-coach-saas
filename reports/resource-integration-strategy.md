# 現有資源整合策略報告

## 一、核心問題回答

### 問題 1：提示詞精簡後，爆款資料庫和向量資料庫是不是就沒用了？

**答案：不是！這些資源仍然非常有價值，只是使用方式需要改變。**

| 資源 | 原本用途 | 精簡後用途 | 價值變化 |
|------|----------|------------|----------|
| **爆款資料庫** | 注入提示詞當範例 | 選題推薦 + 風格參考 | ⬆️ 更有價值 |
| **向量資料庫** | 語意匹配範例 | 選題語意搜尋 + 同質性檢測 | ⬆️ 更有價值 |
| **IP 地基** | 注入提示詞當人設 | 選題過濾 + 風格校準 | ⬆️ 更有價值 |
| **戰報分析** | 被動記錄 | 主動學習 + 策略建議 | ⬆️ 更有價值 |

**關鍵轉變**：從「塞進提示詞」變成「智慧運用」。

## 二、現有資源盤點

### 資源 1：爆款貼文資料庫（viralExamples）

**現有數據**：
- 1,739+ 篇爆款貼文
- 包含：完整內容、讚數、關鍵字、漏斗階段、內容類型
- 已分類：Top200 全領域 + Top20 各關鍵字

**原本用途**：
- 注入提示詞當 Few-Shot 範例
- 問題：範例太多，LLM 變成「模仿」而非「創作」

**精簡後用途**：
1. **選題推薦**：根據用戶領域推薦相關選題
2. **切角分析**：分析同一主題的不同切角
3. **風格參考**：用戶選擇後，提取該貼文的風格特徵（而非直接複製）

### 資源 2：向量資料庫（viralExampleEmbeddings）

**現有數據**：
- 爆款貼文的 Embedding 向量（1536 維）
- 支援語意搜尋

**原本用途**：
- 根據用戶素材，找出語意相似的爆款範例
- 問題：找到的範例直接注入提示詞，產生「抄襲感」

**精簡後用途**：
1. **選題語意搜尋**：用戶輸入想法，找出相關選題
2. **同質性檢測**：檢測生成內容是否與過去內容太相似
3. **切角發現**：找出同一主題的不同切角

### 資源 3：IP 地基（ipProfiles）

**現有數據**：
- 用戶的人設定位（職業、語氣、觀點）
- 英雄旅程四階段
- 情感共鳴語句
- 九宮格內容矩陣

**原本用途**：
- 全部注入提示詞
- 問題：提示詞過長，LLM 無法全部遵守

**精簡後用途**：
1. **選題過濾**：只推薦符合用戶領域的選題
2. **風格校準**：生成後檢查是否符合用戶風格
3. **情感注入**：在具體化引導時，提示用戶的情感連結語句

### 資源 4：戰報分析（postMetrics）

**現有數據**：
- 用戶發布的貼文數據（觸及、互動、讚數）
- 爆文標記和分析
- 自我反思

**原本用途**：
- 被動記錄，偶爾用於策略總結
- 問題：沒有主動學習和應用

**精簡後用途**：
1. **成功模式學習**：分析用戶的爆文特徵
2. **選題建議**：根據過去表現，推薦可能成功的選題
3. **風格優化**：學習用戶的成功風格，應用到生成中

## 四、戰報分析的整合

### 問題：爆款選題庫會參考戰報分析嗎？

**答案：會！而且是非常重要的整合。**

### 整合方式

#### 1. 選題推薦排序

根據用戶過去的戰報數據，調整選題推薦的排序：

```typescript
// 選題推薦邏輯
function rankTopics(topics: Topic[], userMetrics: PostMetric[]) {
  // 分析用戶的爆文特徵
  const viralPosts = userMetrics.filter(m => m.isViral);
  const viralContentTypes = analyzeContentTypes(viralPosts);
  const viralKeywords = analyzeKeywords(viralPosts);
  
  // 根據用戶的成功模式排序
  return topics.sort((a, b) => {
    let scoreA = 0;
    let scoreB = 0;
    
    // 如果選題類型符合用戶的爆文類型，加分
    if (viralContentTypes.includes(a.contentType)) scoreA += 10;
    if (viralContentTypes.includes(b.contentType)) scoreB += 10;
    
    // 如果選題關鍵字符合用戶的爆文關鍵字，加分
    if (viralKeywords.some(k => a.keywords.includes(k))) scoreA += 5;
    if (viralKeywords.some(k => b.keywords.includes(k))) scoreB += 5;
    
    return scoreB - scoreA;
  });
}
```

#### 3. 避免重複選題

根據戰報記錄，避免推薦用戶最近寫過的選題：

```typescript
// 過濾最近寫過的選題
function filterRecentTopics(topics: Topic[], recentPosts: Post[]) {
  const recentKeywords = recentPosts.flatMap(p => extractKeywords(p.body));
  
  return topics.filter(topic => {
    // 如果選題關鍵字與最近貼文重複度太高，過濾掉
    const overlap = topic.keywords.filter(k => recentKeywords.includes(k));
    return overlap.length / topic.keywords.length < 0.5;
  });
}
```

## 五、IP 地基的整合

### 問題：爆款選題庫會提取 IP 地基的資料嗎？

**答案：會！而且是更智慧的運用。**

### 整合方式

#### 1. 選題過濾

根據 IP 地基的領域設定，過濾選題：

```typescript
// 根據 IP 地基過濾選題
function filterByIpProfile(topics: Topic[], ipProfile: IpProfile) {
  const userOccupation = ipProfile.occupation; // 例如：塔羅占卜師
  const userThemes = ipProfile.contentMatrixThemes; // 例如：['感情', '事業', '自我成長']
  
  return topics.filter(topic => {
    // 選題關鍵字必須與用戶領域相關
    return topic.keywords.some(k => 
      userThemes.includes(k) || 
      isRelatedToOccupation(k, userOccupation)
    );
  });
}
```

#### 2. 風格匹配提示

在選題推薦中，標記哪些選題更符合用戶風格：

```typescript
// 計算選題與用戶風格的匹配度
function calculateStyleMatch(topic: Topic, ipProfile: IpProfile) {
  let matchScore = 0;
```

## 六、資源使用方式的轉變

### 總結：從「塞進提示詞」到「智慧運用」

| 資源 | 原本方式 | 新方式 | 效果 |
|------|----------|--------|------|
| **爆款資料庫** | 注入提示詞當範例 | 選題推薦 + 切角分析 | 不再「模仿」，而是「啟發」 |
| **向量資料庫** | 語意匹配後注入 | 選題搜尋 + 同質性檢測 | 不再「抄襲」，而是「發現」 |
| **IP 地基** | 全部注入提示詞 | 選題過濾 + 風格校準 + 情感提示 | 不再「強制」，而是「引導」 |
| **戰報分析** | 被動記錄 | 主動學習 + 推薦排序 | 不再「存檔」，而是「應用」 |

### 核心理念

> **資源的價值不在於「有多少」，而在於「怎麼用」。**

精簡提示詞後，這些資源反而更有價值，因為：
1. **不再塞進提示詞**：避免 LLM 過載
2. **在正確的時機使用**：選題階段、引導階段、校準階段
3. **以用戶為中心**：根據用戶的數據和風格，提供個人化建議

## 八、結論

**提示詞精簡不會讓現有資源失去價值，反而會讓它們發揮更大的作用。**

關鍵轉變：
1. **爆款資料庫**：從「範例注入」變成「選題推薦」
2. **向量資料庫**：從「語意匹配」變成「選題搜尋」
3. **IP 地基**：從「強制注入」變成「智慧引導」
4. **戰報分析**：從「被動記錄」變成「主動學習」

這樣的設計讓系統更像一個「教練」，而不是一個「代筆者」。
