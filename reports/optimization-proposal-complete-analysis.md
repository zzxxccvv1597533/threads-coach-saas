## 一、執行摘要

### 核心問題

目前系統的爆款資料庫（2,542 個向量、29,475 篇貼文、603 個成功因素）**利用時機太晚**。學員在選題時只看到「參考」，不知道這個主題好不好，等到生成全文時才發現「這個主題本來就不容易爆」，已經來不及調整。

### 優化目標

將爆款數據的價值**前置**到選題階段，讓學員在第一步就能做出更好的決策。

### 預期效果

| 指標 | 優化前 | 優化後 |
|------|--------|--------|
| 選題決策信心 | 低（只有參考） | 高（有評分和建議） |
| 內容調整時機 | 生成後才發現問題 | 選題時就能調整 |
| 爆款資料庫利用率 | 30%（只用於參考） | 80%（全流程指導） |

## 三、優化方案詳細說明

### 3.1 優化 A：選題爆款潛力評分（P0）

#### 解決的問題

學員輸入主題後，不知道這個主題好不好，只能「碰運氣」。

#### 優化方案

在腦力激盪階段，為每個主題計算「爆款潛力分數」（0-100 分）。

#### 計算邏輯

```typescript
function calculateViralPotential(topic: string): number {
  // 1. Embedding 相似度（權重 40%）
  const similarExamples = await findSimilarViralExamples(topic, 10);
  const avgSimilarity = average(similarExamples.map(e => e.similarity));
  const similarityScore = avgSimilarity * 100 * 0.4;
  
  // 2. 相似爆款的平均讚數（權重 30%）
  const avgLikes = average(similarExamples.map(e => e.viralExample.likes));
  const likesScore = Math.min(avgLikes / 50, 100) * 0.3; // 5000 讚 = 滿分
  
  // 3. 群集爆文率（權重 20%）
  const cluster = findMatchingCluster(topic);
  const clusterScore = (cluster?.top10Rate || 0) * 100 * 0.2;
  
  // 4. 關鍵字市場數據（權重 10%）
  const keywords = findMatchingKeywords(topic);
  const keywordScore = keywords.length > 0 ? 10 : 0;
  
  return similarityScore + likesScore + clusterScore + keywordScore;
}
```

|------|------|------|----------|
| 分數誤導 | 中 | 學員可能過度依賴分數，忽略內容本質 | 加入說明：「分數僅供參考，好內容才是關鍵」 |
| 計算延遲 | 低 | 需要查詢多個數據源 | 使用快取，預計算常見主題 |
| 分數偏差 | 中 | 新領域可能沒有足夠的爆款參考 | 當相似度低於 50% 時，顯示「數據不足」 |

### 3.2 優化 B：開頭爆款相似度標記（P0）

#### 解決的問題

學員選擇開頭時，不知道哪個更接近爆款，只能「憑感覺」。

#### 優化方案

為每個生成的開頭計算「與爆款的相似度」，並標記「最接近爆款」的選項。

#### 計算邏輯

```typescript
async function rankHooksByViralSimilarity(hooks: string[]): Promise<RankedHook[]> {
  const results = await Promise.all(hooks.map(async (hook) => {
    const embedding = await generateEmbedding(hook);
    const similarExamples = await findSimilarViralExamples(hook, 3);
    const maxSimilarity = Math.max(...similarExamples.map(e => e.similarity));
    const closestExample = similarExamples[0];
    
    return {
      hook,
      similarity: maxSimilarity,
      closestExample: closestExample?.viralExample.opener50,
      closestLikes: closestExample?.viralExample.likes,
    };
  }));
  
  return results.sort((a, b) => b.similarity - a.similarity);
}
```

### 3.3 優化 C：類型智能推薦（P1）

#### 解決的問題

學員選擇內容類型時，不知道「這個主題用哪種類型最容易爆」。

#### 優化方案

根據相似爆款的類型分佈，推薦最適合的內容類型。

#### 計算邏輯

```typescript
async function recommendContentType(topic: string): Promise<TypeRecommendation[]> {
  const similarExamples = await findSimilarViralExamples(topic, 20);
  
  // 統計各類型的分佈
  const typeStats: Record<string, { count: number; avgLikes: number }> = {};
  
  for (const ex of similarExamples) {
    const type = ex.viralExample.contentType || 'unknown';
    if (!typeStats[type]) {
      typeStats[type] = { count: 0, avgLikes: 0 };
    }
    typeStats[type].count++;
    typeStats[type].avgLikes += ex.viralExample.likes || 0;
  }
  
  // 計算平均讚數
  for (const type in typeStats) {
    typeStats[type].avgLikes /= typeStats[type].count;
  }
  
  // 排序並返回推薦
  return Object.entries(typeStats)
    .map(([type, stats]) => ({
      type,
      count: stats.count,
      avgLikes: Math.round(stats.avgLikes),
      recommendation: stats.count >= 5 ? '強烈推薦' : stats.count >= 3 ? '推薦' : '可嘗試',
    }))
    .sort((a, b) => b.count - a.count);
}
```

### 3.4 優化 D：優化建議生成（P1）

#### 解決的問題

學員不知道「如何調整主題更容易爆」。

#### 優化方案

根據相似爆款的特徵，自動生成優化建議。

#### 計算邏輯

```typescript
async function generateOptimizationSuggestions(topic: string): Promise<string[]> {
  const similarExamples = await findSimilarViralExamples(topic, 10);
  const suggestions: string[] = [];
  
  // 分析相似爆款的特徵
  const features = {
    hasNumber: 0,
    hasQuestion: 0,
    hasContrast: 0,
    hasStory: 0,
    avgLength: 0,
  };
  
  for (const ex of similarExamples) {
    const text = ex.viralExample.postText || '';
    if (/\d/.test(text)) features.hasNumber++;
    if (/[？?]/.test(text)) features.hasQuestion++;
    if (/但是|不是|而是/.test(text)) features.hasContrast++;
    if (/我|那天|當時/.test(text)) features.hasStory++;
    features.avgLength += text.length;
  }
  
  features.avgLength /= similarExamples.length;
  
  // 生成建議
  if (features.hasNumber >= 7) {
    suggestions.push(`加入具體數字（如「3 年」「5 分鐘」）可提升爆款率 41%`);
  }
  if (features.hasQuestion >= 6) {
    suggestions.push(`在開頭或結尾加入問句可提升爆款率 49%`);
  }
  if (features.hasContrast >= 5) {
    suggestions.push(`用反常識或對比開頭（如「不是...而是...」）可提升爆款率 28%`);
  }
  if (features.hasStory >= 6) {
    suggestions.push(`用第一人稱故事開頭可提升爆款率 35%`);
  }
  
  return suggestions;
}
```

async function getHighPotentialTopics(userId: string): Promise<TopicSuggestion[]> {

| **計算延遲** | 低 | 中 | 🟢 低 | 使用快取、並行計算 |
| **數據偏差** | 中 | 中 | 🟡 中 | 加入基準線校正、顯示「數據不足」提示 |
| **學員困惑** | 低 | 低 | 🟢 低 | 清晰的 UI 設計、漸進式引導 |

### 4.2 最大風險：內容同質化

#### 問題描述

如果所有學員都：
1. 選擇「爆款潛力最高」的主題
2. 選擇「最接近爆款」的開頭
3. 選擇「推薦」的內容類型

結果會導致所有人的內容越來越像，失去個人特色。

#### 緩解措施

1. **UI 層面**：
   - 不要把「爆款相似度」放在最顯眼的位置
   - 加入「這個開頭最像你的風格」的標記
   - 強調「選擇符合你風格的選項」

2. **算法層面**：
   - 加入「多樣性獎勵」：如果學員最近都選高相似度的選項，降低高相似度選項的排名
   - 加入「個人風格匹配」：根據學員過去的選擇，推薦符合其風格的選項

3. **教育層面**：
   - 在 UI 中加入提示：「爆款參考只是起點，你的獨特視角才是關鍵」
   - 定期提醒學員：「過度模仿會失去個人特色」

### 6.2 新增利用場景

#### 場景 1：選題前的「今日推薦」

**數據來源**：群集 + 近期爆款 + 用戶領域

**利用方式**：
1. 識別用戶領域（從 IP 地基）
2. 取得該領域的高爆款率群集
3. 從每個群集取得近期爆款範例
4. 推薦給用戶

#### 場景 2：選題時的「爆款潛力評分」

**數據來源**：Embedding + 群集 + 關鍵字

**利用方式**：
1. 計算 Embedding 相似度（權重 40%）
2. 查詢群集爆文率（權重 30%）
3. 查詢關鍵字市場數據（權重 20%）
4. 計算相似爆款平均讚數（權重 10%）
5. 綜合計算爆款潛力分數

#### 場景 3：選題時的「優化建議」

**數據來源**：相似爆款的特徵分析

**利用方式**：
1. 取得 10 篇相似爆款
2. 分析特徵分佈（數字、問句、對比、故事）
3. 找出「高於平均」的特徵
4. 生成優化建議

#### 場景 4：類型選擇時的「智能推薦」

**數據來源**：相似爆款的類型分佈

**利用方式**：
1. 取得 20 篇相似爆款
2. 統計各類型的數量和平均讚數
3. 排序並推薦

#### 場景 5：開頭選擇時的「爆款相似度」

**數據來源**：Embedding 匹配

**利用方式**：
1. 為每個生成的開頭計算 Embedding
2. 與爆款向量庫比對
3. 標記最接近爆款的選項

   - 預估時間：5-7 小時

   - 這個功能解決「不知道發什麼」的問題

1. **UI 設計**：
   - 不要讓數據壓過內容本身
