> 基於第一性原理重新審視系統，解決 AI 感問題並設計選題輔助功能

## 一、核心問題診斷

### 1.1 AI 感問題的根本原因

經過深入分析現有系統的程式碼和提示詞架構，我發現 AI 感問題的根源**不是缺少去 AI 痕跡的規則**，而是系統設計偏離了創作的第一性原理。

| 問題層級 | 具體問題 | 影響 |
|----------|----------|------|
| **架構層** | 提示詞過長（約 4,000 字） | LLM 選擇性執行，產生拼湊感 |
| **規則層** | 規則互相矛盾（口語 vs 結構） | 輸出變得不自然 |
| **範例層** | Few-Shot 學到句式而非風格 | 每篇都像同一個模板 |
| **流程層** | 缺少具體素材的引導 | AI 被迫編造細節 |

### 1.2 A/B 測試的關鍵發現

我用「創作者漲價焦慮」的素材進行了三組測試：

| 版本 | 提示詞長度 | AI 感類型 | 問題 |
|------|------------|-----------|------|
| A（現有系統） | 2,500 字 | 公式化 AI 感 | 結構太工整、結尾太制式 |
| B（精簡提示詞） | 400 字 | 教學文 AI 感 | 像在上課、說理太多 |
| C（加入 IP 地基） | 400 字 | 雞湯文 AI 感 | 「親愛的」過多、太溫暖 |
| **D（IP 地基 + 具體故事）** | 400 字 | **最低** | 有具體數字和對話 |

**關鍵發現**：

> **AI 感的高低，與提示詞的長度關係不大，與「素材的具體程度」高度相關。**

當素材包含：
- 具體數字（1,800 → 3,600 元）
- 具體時間（上週）
- 具體對話（「萬一客戶都跑光了怎麼辦？」）

AI 就不需要編造細節，生成的內容自然有「人味」。

## 三、解決方案：兩大核心改進

基於以上分析，我提出兩個核心改進方向：

### 3.1 方向一：具體化引導（解決 AI 感問題）

**核心理念**：問題不在 AI，在於輸入不夠具體。

#### 3.1.1 故事型的具體化引導

**現有設計**：
```
素材/主題：[                                    ]
placeholder: "例如：最近很多人問我關於自我懷疑的問題..."
```

**優化後設計**：
```
📖 故事型需要具體細節才能有「人味」

1. 什麼時候發生的？
   [  上週  ] [  前幾天  ] [  昨天  ] [  自訂  ]

2. 主角是誰？
   [  學員/案主  ] [  朋友  ] [  自己  ] [  自訂  ]

3. 具體數字（如果有的話）
   例如：金額、時間、次數
   [                                    ]

4. 他/她說了什麼？（原話）
   例如：「老師，萬一客戶都跑光了怎麼辦？」
   [                                    ]

5. 你的回應或感悟
   [                                    ]
```

#### 3.1.2 各貼文類型的優化方向

| 貼文類型 | 現有欄位 | 新增引導 |
|----------|----------|----------|
| **故事型** | material | 時間 + 人物 + 數字 + 對話 + 感悟 |
| **觀點型** | stance, reason | 新增：支撐觀點的具體案例 |
| **反差型** | common_belief, truth | 新增：你怎麼發現這個真相的？ |
| **對話型** | question, context | 強調：對話要用「原話」 |
| **閒聊型** | topic | 新增：今天發生了什麼具體的事？ |

#### 3.2.2 選題推薦邏輯

```typescript
// 選題推薦的核心邏輯

interface TopicSuggestion {
  topic: string;           // 選題標題
  hookType: string;        // 建議的開頭類型
  contentType: string;     // 建議的貼文類型
  relatedViral: {          // 相關爆款
    opener: string;
    likes: number;
  };
  guidedQuestions: string[]; // 引導問題
}

async function getTopicSuggestions(
  userDomain: string,      // 用戶領域（從 IP 地基取得）
  recentThoughts?: string  // 用戶最近的想法（可選）
): Promise<TopicSuggestion[]> {
  
  // 1. 從爆款資料庫中找出與用戶領域相關的高讚爆款
  const relatedVirals = await findSimilarViralExamples(
    userDomain, 
    30,  // 取 30 篇
    undefined,
    true // 使用語意擴展
  );
  
  // 2. 從爆款中提取「可複製的選題方向」
  const topicDirections = extractTopicDirections(relatedVirals);
  
  // 3. 根據用戶的 IP 地基，過濾出最適合的選題
  const filteredTopics = filterByUserProfile(topicDirections, userDomain);
  
  // 4. 為每個選題生成「引導問題」
  const suggestionsWithGuides = filteredTopics.map(topic => ({
    ...topic,
    guidedQuestions: generateGuidedQuestions(topic.contentType),
  }));
  
  return suggestionsWithGuides;
}
```

#### 3.2.3 引導問題設計

當用戶選擇一個選題後，系統會根據選題類型，提供「引導問題」幫助用戶回想具體細節：

| 選題類型 | 引導問題 |
|----------|----------|
| **故事型** | 「這件事是什麼時候發生的？」「當時對方說了什麼？」「你當時的感受是什麼？」 |
| **觀點型** | 「你為什麼會有這個觀點？」「有沒有具體的例子支撐？」「反對的人可能會說什麼？」 |
| **反差型** | 「大家通常怎麼想？」「你發現的真相是什麼？」「你是怎麼發現的？」 |
| **對話型** | 「是誰問你這個問題？」「他問這個問題的背景是什麼？」「你怎麼回答的？」 |

## 四、AI 去痕的精準策略

### 4.1 Humanizer-zh 規則的選擇性整合

經過分析，我建議**只整合以下規則**：

| 規則 | 適用 | 原因 |
|------|------|------|
| **AI 詞彙替換** | ✅ | 「此外」「至關重要」等詞確實有 AI 感 |
| **節奏變化檢測** | ✅ | 連續三句長度相同確實不自然 |
| **信任讀者原則** | ✅ | 不要繞圈子，直接說 |

**不建議整合的規則**：

| 規則 | 不適用 | 原因 |
|------|--------|------|
| 刪除金句 | ❌ | Threads 爆款需要可截圖金句 |
| 打破三段式 | ❌ | Threads 清單型需要三點 |
| 刪除所有填充短語 | ❌ | Threads 需要「呼吸感」 |

### 4.2 AI 詞彙替換清單

```typescript
// 需要替換的 AI 詞彙

export const AI_WORDS_TO_REPLACE = {
  // 連接詞
  '此外': ['還有', '另外', '對了'],
  '然而': ['但', '不過', '可是'],
  '因此': ['所以', '這樣一來'],
  '值得注意的是': ['有趣的是', '我發現'],
  
  // 強調詞
  '至關重要': ['很重要', '超重要', '關鍵'],
  '深入探討': ['聊聊', '說說', '來談談'],
  '不可或缺': ['一定要有', '少不了'],
  
  // 結尾詞
  '總而言之': ['總之', '說到底'],
  '綜上所述': ['所以啊', '講這麼多'],
};
```

### 4.3 結尾多樣化

**問題**：目前生成的內容結尾常常是「你們覺得呢？」「留言告訴我」。

**解決方案**：建立 CTA 類型庫，根據內容類型隨機選擇。

```typescript
export const CTA_TYPES = {
  // 問句型（適合觀點型、反差型）
  question: [
    '你也有過這種經驗嗎？',
    '你怎麼看？',
    '你會怎麼選？',
  ],
  
  // 陳述型（適合故事型）
  statement: [
    '就這樣，跟你分享一下。',
    '這件事讓我想了很久。',
    '希望對你有幫助。',
  ],
  
  // 反思型（適合知識型）
  reflection: [
    '我自己也還在學習中。',
    '這是我目前的理解，可能還不夠完整。',
  ],
  
  // 無 CTA（適合閒聊型）
  none: [
    '', // 直接結束，不加 CTA
  ],
};
```

## 七、結論

### 7.1 核心洞察

> **AI 感的問題，不是 AI 的問題，是輸入的問題。**

當創作者提供具體的素材（時間、人物、數字、對話），AI 就不需要編造細節，生成的內容自然有「人味」。

### 7.2 系統定位的轉變

| 現有定位 | 新定位 |
|----------|--------|
| 幫用戶寫爆款 | 幫用戶說出想說的話 |
| 套用爆款公式 | 引導用戶回想具體細節 |
| AI 主導生成 | 用戶主導內容，AI 輔助表達 |
