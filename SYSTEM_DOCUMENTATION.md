## 一、系統概述

幕創行銷 Threads AI 教練是一套專為 Threads 平台內容創作者設計的 SaaS 系統，協助用戶建立個人品牌、產出高品質內容、追蹤成效並實現變現。系統核心理念是「讓每個人都能寫出有人味的爆款內容」，透過 AI 輔助但保留個人特色。

## 三、核心功能模組

### 3.1 IP 地基模組

IP 地基是整個系統的基礎，儲存用戶的個人品牌定位資料，所有 AI 生成都會參考這些資料。

#### 3.1.1 資料結構

| Tab | 欄位 | 說明 | AI 使用方式 |
|-----|------|------|-------------|
| **基本資料** | 職業/身份 | 用戶的專業身份 | 決定寫作視角 |
| | 說話風格 | 語氣調性 | 決定文案語氣 |
| | 核心信念 | 價值觀宣言 | 支撐觀點內容 |
| | 身份標籤 | 多個標籤 | 增加共鳴點 |
| **人設三支柱** | 專業權威 | 專業背景 | 展現專業度 |
| | 情感共鳴 | 共情能力 | 建立連結 |
| | 獨特觀點 | 差異化觀點 | 差異化定位 |
| **我的故事** | 緣起 | 為什麼開始 | 故事型內容開頭 |
| | 過程 | 遇到什麼困難 | 展現同理心 |
| | 英雄登場 | 轉折點 | 建立信任 |
| | 結局與使命 | 現在的使命 | 變現內容 CTA |
| **內容矩陣** | 三大受眾 | 核心/潛在/機會 | 決定內容方向 |
| | 痛點矩陣 | 每受眾 3 個痛點 | 選題靈感來源 |
| | 內容支柱 | 3 大主題 | 內容分類 |
| **我的風格** | 爆款貼文 | 3-10 篇範例 | AI 學習風格 |
| | 風格分析 | AI 分析結果 | 生成時模仿 |

#### 3.1.2 資料流向

```
用戶輸入 IP 地基資料
        ↓
儲存到 ipProfiles 表
        ↓
發文工作室生成時讀取
        ↓
注入 AI 提示詞
        ↓
生成符合用戶風格的內容
```

### 3.2 發文工作室模組

發文工作室是內容生成的核心，提供兩種模式：「沒靈感」和「有素材」。

#### 3.2.1 沒靈感流程

這是一個 8 步驟的引導式流程：

| 步驟 | 功能 | 資料讀取 | 產出 |
|------|------|----------|------|
| Step 1 | AI 給主題建議 | IP 地基、內容矩陣 | 3 個主題選項 |
| Step 2 | 選擇貼文類型 | 內容類型定義 | 選定類型 |
| Step 3 | 選擇 Hook 風格 | Hook 策略庫 | 選定風格 |
| Step 4 | 補充關鍵資訊 | 動態欄位 | 用戶輸入 |
| Step 5 | AI 生成 Hook | IP 地基 + 用戶風格 | 3 個 Hook 選項 |
| Step 6 | 生成全文 | 所有 IP 資料 | 完整草稿 |
| Step 7 | AI 對話修改 | 對話歷史 | 修改後草稿 |
| Step 8 | 人味潤飾 | 用戶風格分析 | 最終版本 |

#### 3.2.2 有素材流程

用戶已有素材（逐字稿、靈感、事件）時的快速生成流程：

| 步驟 | 功能 | 資料讀取 | 產出 |
|------|------|----------|------|
| Step 1 | 貼入素材 | - | 用戶輸入 |
| Step 2 | 選擇內容類型 | 內容類型定義 | 選定類型 |
| Step 3 | AI 生成草稿 | IP 地基 + 用戶風格 | 完整草稿 + 診斷結果 |
| Step 4 | AI 對話修改 | 對話歷史 | 修改後草稿 |

#### 3.2.3 內容類型定義

系統支援 10 種情緒內容類型：

| 類型 | 說明 | 適用場景 |
|------|------|----------|
| knowledge | 乾貨知識 | 教學、分享專業 |
| summary | 懶人包 | 整理資訊 |
| story | 故事 | 個人經歷 |
| viewpoint | 觀點 | 表達立場 |
| contrast | 對比 | 比較分析 |
| casual | 閒聊 | 日常分享 |
| dialogue | 對話 | 模擬對話 |
| question | 提問 | 引發討論 |
| poll | 投票 | 互動調查 |
| quote | 金句 | 精華語錄 |

以及 8 種變現內容類型：

|------|------|----------|

### 3.3 AI 生成邏輯

#### 3.3.1 提示詞結構

AI 生成時會組合以下資訊：

```
=== 創作者 IP 地基（必須在內容中展現） ===
【你的身份】{職業}
【說話風格】{風格}
【人設三支柱】
  • 專業權威：{內容}
  • 情感共鳴：{內容}
  • 獨特觀點：{內容}
【核心信念】{信念}
【英雄旅程故事】{四階段故事}
【身份標籤】{標籤}

=== 目標受眾 ===
{受眾描述}

=== 內容支柱 ===
{三大主題}

=== 用戶寫作風格 ===
【語氣特徵】{風格}
【常用句式】{句式}
【口頭禪】{口頭禪}
【Hook 風格偏好】{偏好}
【比喻風格】{風格}
【情緒節奏】{節奏}
【爆款元素】{元素}

=== 內容類型 ===
類型：{選擇的類型}
說明：{類型說明}

=== Threads 爆款風格規則 ===
{排版規則、口語化規則、禁止詞彙}
```

#### 3.3.2 四透鏡框架

所有生成內容都會經過四透鏡檢核：

| 透鏡 | 檢核問題 | 標準 |
|------|----------|------|
| 心法透鏡 | 傳遞渴望還是焦慮？ | 必須是渴望導向 |
| 人設透鏡 | 像不像你說的話？ | 必須有個人特色 |
| 結構透鏡 | 好不好吸收？ | 結構清晰有邏輯 |
| 轉化透鏡 | 讀者看完要做什麼？ | 有明確 CTA |

#### 3.3.3 Hook 策略

系統使用三大心理學原理生成 Hook：

| 原理 | 策略 | 句型範例 |
|------|------|----------|
| 鏡像效應 | 鏡像策略 | 「你是不是也...」「有沒有人也會這樣...」 |
| 認知失調 | 反差策略 | 「我以為...但其實...」「我不是...但我還是...」 |
| 好奇心缺口 | 解法策略 | 「...有問題？這幾點先看懂」 |

#### 3.3.4 生成後診斷

每次生成後會自動產出診斷結果：

| 類別 | 檢查項目 | 顯示方式 |
|------|----------|----------|
| 優勢 | Hook 強度、翻譯機、CTA | 綠色標籤 |
| 可加強 | 身分標籤、排版、轉折詞 | 黃色標籤 |

### 3.4 文案健檢模組

提供已發布或草稿內容的 AI 診斷與優化。

#### 3.4.1 五維度評分

| 維度 | 權重 | 評分標準 |
|------|------|----------|

| CTA 互動召喚 | 10% | 是否有效召喚行動 |
| 四透鏡檢核 | 30% | 心法/人設/結構/轉化 |

#### 3.4.2 功能列表

| 功能 | 說明 | 輸入 | 輸出 |
|------|------|------|------|
| 快速診斷 | 五維度評分 | 文案內容 | 分數 + 建議 |
| 一鍵優化 | AI 自動修改 | 文案內容 | 優化後版本 |
| 對話修改 | 指定修改 | 文案 + 指令 | 修改後版本 |

### 4.3 用戶風格學習流程

```
用戶貼入 3-10 篇爆款貼文
        ↓
儲存到 userWritingStyles.samplePosts
        ↓
呼叫 analyzeUserStyle API
        ↓
AI 分析提取風格特徵：
  - 語氣特徵（溫暖/犀利/幽默）
  - 常用句式
  - 口頭禪
  - Hook 風格偏好
  - 比喻風格
  - 情緒節奏
  - 爆款元素
        ↓
儲存分析結果到 userWritingStyles 表
        ↓
後續生成時自動讀取並注入提示詞
```

## 七、Threads 爆款風格規則

系統內建的 Threads 爆款風格規則，確保生成內容符合平台特性：

### 7.1 字數限制

| 內容類型 | 字數範圍 |
|----------|----------|
| 一般貼文 | 300-500 字 |
| 短貼文 | 150-250 字 |
| 變現貼文 | 400-600 字 |

### 7.2 呼吸感排版

| 規則 | 說明 |
|------|------|
| 段落結構 | 每 2-4 行為一個段落 |
| 空行規則 | 段落之間空一行 |
| 句子長度 | 每句 10-15 字，最多 20 字 |

### 7.3 口語化原則

| 原則 | 範例 |
|------|------|
| 傳訊息感 | 像在 LINE 跟朋友聊天 |
| 省略主詞 | 「真的超累」而非「我真的超累」 |
| 不完整句 | 「結果呢？」「就這樣。」 |
| 語助詞 | 「真的」「超」「欹」「啊」「吧」「呢」「啦」 |
| 口語表達 | 「說真的」「老實說」「不騙你」 |

### 7.4 禁止詞彙

| 類別 | 禁止詞彙 |
|------|----------|
| AI 常用詞 | 「讓我們」「一起來」「今天要分享」「分享一下」 |
| 制式開頭 | 「親愛的朋友們」「各位」「大家好」 |
| 結構詞 | 「首先」「其次」「最後」「第一」「第二」「第三」 |
| 雞湯結尾 | 「希望對你有幫助」「加油！」「你可以的！」 |
| 格式符號 | Markdown 符號、條列式 |

## 十、系統特色總結

1. **個人化 AI**：所有生成都基於用戶的 IP 地基，確保內容有個人特色

3. **四透鏡框架**：確保內容品質的檢核機制

5. **完整工作流**：從靈感到發布的完整流程支援
6. **數據追蹤**：戰報分析幫助優化內容策略
