# 爆款驗證結果與用戶風格學習機制設計

## 一、爆款資料庫驗證結果

### 分析數據（1,240 篇爆款）

| 指標 | 數值 | 說明 |
|------|------|------|
| 平均字數 | 109 字 | 比預期短，Threads 偏好短文 |
| 平均段落數 | 1 段 | 大多是單段或少量分段 |
| 平均句子長度 | 16 字 | 短句為主 |
| 有 Hook 開頭 | 6% | 大多數爆款不需要刻意 Hook |
| 有 CTA 結尾 | 1% | 幾乎沒有爆款用 CTA |
| 有問句 | 21% | 約五分之一有問句 |

### 語氣詞使用 Top 10

| 排名 | 語氣詞 | 出現次數 |
|------|--------|---------|
| 1 | 好 | 668 |
| 2 | 很 | 564 |
| 3 | 真的 | 215 |
| 4 | 太 | 209 |
| 5 | 因為 | 154 |
| 6 | 超 | 143 |
| 7 | 就是 | 138 |
| 8 | 啊 | 109 |
| 9 | 欸 | 96 |
| 10 | 所以 | 93 |

### 需要避免的用詞

| 類型 | 出現率 | 說明 |
|------|--------|------|
| 教學語氣詞 | 50% | 這個比例偏高，但爆款中也有使用 |
| 行銷術語 | 1% | 幾乎沒有爆款用行銷術語 |
| 書面情緒詞 | 0% | 完全沒有爆款用「心裡一沉」這類詞 |
| 刻意比喻 | 5% | 很少爆款用刻意比喻 |

### 驗證結論

**✅ 優化方向正確**：
1. **口語化是對的**：爆款大量使用「欸」「啊」「喔」「啦」等語氣詞
2. **短句是對的**：平均句子長度只有 16 字
3. **不要 CTA 是對的**：只有 1% 的爆款有 CTA
4. **不要書面情緒詞是對的**：0% 使用率
5. **不要刻意比喻是對的**：只有 5% 使用率

**⚠️ 需要調整的認知**：
1. **Hook 不是必須**：只有 6% 的爆款有明確 Hook，大多數是直接講故事
2. **教學語氣不是絕對禁止**：50% 的爆款有使用，但要用口語化的方式

## 三、用戶風格學習機制設計

### 核心理念

> 讓 AI 從用戶上傳的貼文中學習，產出越來越像用戶自己的風格

### 資料來源

| 來源 | 說明 | 學習內容 |
|------|------|----------|
| **戰報分析** | 用戶上傳的歷史貼文 | 用詞習慣、句式結構、語氣詞偏好 |
| **個人風格** | 用戶設定的口頭禪、常用詞 | 特定詞彙、開場白、結尾習慣 |
| **對話紀錄** | 用戶與 AI 的對話修改歷史 | 用戶偏好的修改方向 |

### 風格特徵萃取

從用戶上傳的貼文中，萃取以下特徵：

```typescript
interface UserStyleProfile {
  // 基本統計
  avgLength: number;           // 平均字數
  avgSentenceLength: number;   // 平均句子長度
  avgParagraphs: number;       // 平均段落數
  
  // 語氣詞偏好
  toneWords: {
    word: string;
    frequency: number;         // 使用頻率（每 100 字出現幾次）
  }[];
  
  // 開頭模式偏好
  openerPatterns: {
    pattern: string;           // 如「直接對話」「自嘲開場」「時間點」
    frequency: number;
  }[];
  
  // 結尾模式偏好
  closerPatterns: {
    pattern: string;           // 如「疑問句」「自嘲」「開放式」
    frequency: number;
  }[];
  
  // 特殊用詞
  signatureWords: string[];    // 用戶的招牌用詞
  avoidWords: string[];        // 用戶不喜歡的詞
  
  // 結構偏好
  usesDialogue: boolean;       // 是否常用對話體
  usesEmoji: boolean;          // 是否常用 emoji
  usesHashtag: boolean;        // 是否常用 hashtag
}
```

### 學習流程

```
1. 用戶上傳貼文（戰報分析 / 個人風格）
   ↓
2. 系統萃取風格特徵
   ↓
3. 儲存到用戶的 StyleProfile
   ↓
4. 生成內容時，將 StyleProfile 注入提示詞
   ↓
5. AI 根據 StyleProfile 調整輸出風格
   ↓
6. 用戶修改內容，系統學習修改偏好
   ↓
7. 更新 StyleProfile（持續學習）
```

### 提示詞注入方式

```typescript
// 在生成內容時，加入用戶風格指引
const styleGuidance = `
## 用戶風格指引

這位用戶的寫作風格特徵：
- 平均字數：${profile.avgLength} 字
- 常用語氣詞：${profile.toneWords.slice(0, 5).map(t => t.word).join('、')}
- 開頭偏好：${profile.openerPatterns[0]?.pattern || '無特定'}
- 結尾偏好：${profile.closerPatterns[0]?.pattern || '無特定'}
- 招牌用詞：${profile.signatureWords.join('、') || '無'}
${profile.usesDialogue ? '- 喜歡用對話體' : ''}
${profile.usesEmoji ? '- 喜歡用 emoji' : ''}

請模仿這位用戶的風格生成內容，但不要過度刻意。
`;
```

### 適度應用原則

> **重要**：風格學習要「適度應用」，不能每篇都套用相同元素

```typescript
// 隨機決定是否套用某個風格元素
function shouldApplyStyleElement(frequency: number): boolean {
  // frequency 是用戶使用該元素的頻率（0-1）
  // 乘以 0.7 避免過度套用
  return Math.random() < frequency * 0.7;
}
```

### 資料庫設計

```sql
-- 用戶風格檔案
CREATE TABLE user_style_profiles (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id VARCHAR(64) NOT NULL,
  
  -- 基本統計
  avg_length INT,
  avg_sentence_length INT,
  avg_paragraphs INT,
  
  -- JSON 欄位儲存複雜結構
  tone_words JSON,           -- 語氣詞偏好
  opener_patterns JSON,      -- 開頭模式偏好
  closer_patterns JSON,      -- 結尾模式偏好
  signature_words JSON,      -- 招牌用詞
  avoid_words JSON,          -- 避免用詞
  
  -- 結構偏好
  uses_dialogue BOOLEAN DEFAULT FALSE,
  uses_emoji BOOLEAN DEFAULT FALSE,
  uses_hashtag BOOLEAN DEFAULT FALSE,
  
  -- 學習來源統計
  posts_analyzed INT DEFAULT 0,
  last_updated TIMESTAMP,
  
  UNIQUE KEY (user_id)
);

-- 用戶上傳的貼文（用於學習）
CREATE TABLE user_posts_for_learning (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id VARCHAR(64) NOT NULL,
  content TEXT NOT NULL,
  likes INT,
  post_date TIMESTAMP,
  source VARCHAR(32),        -- 'battle_report' | 'personal_style' | 'manual'
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 四、總結

### 爆款驗證結論

| 優化方向 | 驗證結果 |
|---------|---------|
| 口語化（語氣詞） | ✅ 正確，爆款大量使用 |
| 短句 | ✅ 正確，平均 16 字 |
| 不要 CTA | ✅ 正確，只有 1% 使用 |
| 不要書面情緒詞 | ✅ 正確，0% 使用 |
| 不要刻意比喻 | ✅ 正確，只有 5% 使用 |
| Hook 必須 | ⚠️ 需調整，只有 6% 有明確 Hook |

### 用戶風格學習機制

1. **資料來源**：戰報分析、個人風格、對話修改歷史
2. **萃取特徵**：語氣詞、開頭模式、結尾模式、招牌用詞
3. **注入方式**：生成時將 StyleProfile 加入提示詞
4. **適度原則**：不過度套用，保持自然
