## 五、LLM 分級使用的實作複雜度評估

### 答案：不會讓後台執行變得很麻煩

**原因**：系統已經有統一的 LLM 調用層（`invokeLLM`），只需新增一個參數即可切換模型。

**實作方式**：
```typescript
// 現有調用方式（不變）
await invokeLLM({ messages: [...] });

// 分級使用（新增 model 參數）
await invokeLLM({ messages: [...], model: 'pro' });  // 正文生成
await invokeLLM({ messages: [...], model: 'flash' }); // 開頭生成
```

**影響範圍**：
- 只需修改 `server/_core/llm.ts` 加入模型選擇邏輯
- 各個調用點可選擇性加入 model 參數
- 不影響現有功能
