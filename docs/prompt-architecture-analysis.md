# 提示詞架構分析報告

## 一、現有提示詞架構總覽

目前系統的提示詞採用**三層架構**：

| 層級 | 內容 | 字數估計 | 性質 |
|------|------|----------|------|
| **第一層** | 通用規則（口語化、呼吸感、禁止詞） | ~500 字 | ✅ 規則（可保留） |
| **第二層** | 類型規則（字數、結構模板、CTA） | ~400 字 | ⚠️ 公式 + 選項混合 |
| **第三層** | 關鍵字規則（爆款開頭、Few-Shot 範例） | ~1,500 字 | ❌ 強制選項 |
| **最終指示** | 檢查清單、強制格式 | ~600 字 | ❌ 強制選項 |

**總計約 3,000-8,000 字**（視是否有 Few-Shot 範例而定）

---

## 二、「給公式」vs「給選項」的具體差異

### 問題 1：開頭模式 - 給公式但強制選擇

**現有做法（給選項）**：
```
本次必須使用「冒號斷言」格式
格式說明：第一行使用「主題：觀點」的格式
```

**問題**：LLM 沒有選擇權，被強制使用特定格式，導致：
- 同質性高（每次都是「XX 的真相：...」）
- 不一定適合素材
- 缺乏創意空間

**建議做法（給公式讓 LLM 發揮）**：
```
參考以下開頭公式，選擇最適合素材的方式自然開場：

【高效開頭公式】（選擇一種，不要刻意套用）
- 冒號斷言：[主題]的[真相/關鍵/本質]：[反直覺觀點]
- 情緒爆發：我真的[強烈情緒動詞][感受]
- 時間點：[時間詞]我[發現/想通/遇到][事件]
- 家庭故事：[家人稱謂][時間詞][動作/說了什麼]
- 自白坦承：我以前也是[負面行為/狀態]

根據素材內容自然選擇，不要刻意套用。
```

---

### 問題 2：結構模板 - 給固定步驟

**現有做法（給選項）**：
```
結構模板：
第一行：直接拋出觀點（用冒號斷言或情緒爆發格式）
第二段：用 1-2 個經歷或論點支撐（不要超過 3 句）
第三段：強化觀點或轉折
結尾：「你怎麼看？」或「你同意嗎？」
```

**問題**：
- 結構太固定（Hook → 情境 → 經驗 → 價值 → CTA）
- 結尾強制 CTA，不自然
- 段落數固定，缺乏彈性

**建議做法（給公式讓 LLM 發揮）**：
```
【結構參考】（不是固定步驟）
- 節奏：開場 → 展開 → 收尾
- 段落數：根據內容自然安排（3-5 段皆可）
- 每段 2-4 行，保持呼吸感

【結尾方式】（選擇一種，不要每次都問問題）
- 觀點收尾：「這兩個根本不一樣吧」
- 疑問收尾：「你怎麼看？」
- 留白收尾：「就這樣」
- 情緒收尾：「真的很煩」
```

---

### 問題 3：Few-Shot 範例 - 給模仿對象

**現有做法（給選項）**：
```
【風格參考範例】（學習風格，禁止複製）
--- 範例 1（4,200 讚）---
學習的真相：不是你不夠努力...
```

**問題**：
- 即使說「禁止複製」，LLM 仍會模仿結構
- 範例太具體，限制創意
- 範例風格可能與用戶 IP 不符

**建議做法（給公式讓 LLM 發揮）**：
```
【氛圍參考】（感受說話方式，不要模仿結構）
Threads 爆款的共同特徵：
- 像傳訊息給朋友，不是寫文章
- 有情緒起伏，不是平鋪直敘
- 說人話，不用專業術語
- 有呼吸感，不是一大段文字

不需要模仿任何範例，用你自己的方式說話。
```

---

### 問題 4：檢查清單 - 給強制要求

**現有做法（給選項）**：
```
【檢查清單】
□ 第一行是否使用了「冒號斷言」格式？
□ 第一行是否獨立成段（後面空一行）？
□ 字數是否在範圍內？
□ 結尾是否有互動引導？
```

**問題**：
- 變成「填空題」而非「創作」
- LLM 會逐條檢查，失去自然感
- 太多限制反而降低品質

**建議做法（給公式讓 LLM 發揮）**：
```
【品質標準】
- 開頭要有吸引力（不限定格式）
- 內容要有呼吸感
- 結尾要自然（不一定要問問題）
- 整體像在跟朋友聊天
```

## 三、提示詞長度與成效的關係

### 研究發現

根據多項研究 [1] [2] [3]：

| 提示詞長度 | 效果 | 原因 |
|-----------|------|------|
| **< 50 字** | 產出泛泛、缺乏深度 | 資訊不足 |
| **150-300 字** | 最佳效果 | 足夠的上下文，不會干擾 |
| **500+ 字** | 效果下降 | 資訊過載，模型分心 |
| **1,000+ 字** | 明顯下降 | 「Context Rot」現象 |

> "For moderately complex tasks, the optimal prompt length typically falls between 150 and 300 words… without crossing the 500-word threshold." [4]

> "Shorter prompts (under 50 words) led to better performance across all models tested. Longer prompts (150+ words) significantly increased error rates." [5]

### 現有系統的問題

目前系統的提示詞長度：

| 情境 | 估計字數 | 問題 |
|------|----------|------|
| 無關鍵字匹配 | ~2,000 字 | 超過最佳範圍 |
| 有關鍵字匹配 | ~4,000 字 | 嚴重超標 |
| 有 Few-Shot 範例 | ~6,000-8,000 字 | 極度超標 |

**具體影響**：
1. **回應延遲增加**：每增加 500 tokens，延遲增加約 25ms [6]
2. **推理能力下降**：長提示詞會「稀釋」重要指令 [7]
3. **同質性增加**：太多範例反而讓 LLM 模仿而非創作
4. **成本增加**：輸入 tokens 越多，費用越高

## 四、優化建議

### 建議 1：精簡三層架構

| 層級 | 現有字數 | 建議字數 | 調整方式 |
|------|----------|----------|----------|
| 第一層 | ~500 字 | ~200 字 | 保留核心禁止詞，移除冗餘說明 |
| 第二層 | ~400 字 | ~150 字 | 只保留字數範圍和核心特徵 |
| 第三層 | ~1,500 字 | ~100 字 | 移除 Few-Shot 範例，只給公式 |
| 最終指示 | ~600 字 | ~50 字 | 移除檢查清單，只給品質標準 |
| **總計** | **~3,000 字** | **~500 字** | **減少 83%** |

### 建議 2：從「給選項」改為「給公式」

**Before（給選項）**：
```
本次必須使用「冒號斷言」格式
第一行使用「主題：觀點」的格式，冒號後直接給結論

【範例】
學習的真相：不是你不夠努力
經營自己的關鍵：先搞清楚這件事
```

**After（給公式）**：
```
開頭公式參考（自然選擇，不要刻意）：
- [主題]的[真相/關鍵]：[觀點]
- 我真的[情緒][感受]
- [時間]我[發現][事件]
```

### 建議 3：移除 Few-Shot 範例

**理由**：
1. Few-Shot 範例佔用大量 tokens
2. 會導致 LLM 模仿結構
3. 與用戶 IP 風格可能不符

**替代方案**：
- 使用用戶自己的爆款貼文作為風格參考
- 只在「我的風格」Tab 有上傳時才注入

### 建議 4：動態調整提示詞長度

```typescript
// 根據任務複雜度調整
if (isSimpleTask) {
  // 簡單任務：只給核心規則
  promptLength = 200;
} else if (hasUserStyle) {
  // 有用戶風格：注入風格，省略通用範例
  promptLength = 400;
} else {
  // 複雜任務：完整但精簡的提示詞
  promptLength = 500;
}
```

## 五、總結

| 項目 | 現狀 | 建議 |
|------|------|------|
| **提示詞長度** | 3,000-8,000 字 | 300-500 字 |
| **開頭模式** | 強制選擇一種 | 給公式讓 LLM 選擇 |
| **結構模板** | 固定 5 步驟 | 給節奏參考 |
| **Few-Shot 範例** | 3-5 個完整範例 | 移除或用用戶自己的 |
| **檢查清單** | 7 項強制檢查 | 3 項品質標準 |
| **結尾 CTA** | 強制問問題 | 自然收尾 |

**預期效果**：
1. 減少同質性
2. 提高創意空間
3. 降低延遲和成本
4. 更符合用戶 IP 風格
