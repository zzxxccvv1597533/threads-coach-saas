## 一、升級方案總覽

### 1.1 核心設計理念

本次升級的核心理念是將 AI 的角色從「代筆者」轉變為「教練」。這意味著 AI 不再直接為用戶產出完整的貼文內容，而是透過引導、提問、建議的方式，協助用戶發展自己的創作能力。

| 面向 | 代筆模式（現有） | 教練模式（升級後） |
|------|------------------|-------------------|
| AI 角色 | 幫你寫完整貼文 | 引導你思考，你決定要寫什麼 |
| IP 資料使用 | 自動注入到內容中 | 作為「參考」讓你自己決定要不要用 |
| 專業導入 | 系統決定要不要加 | 教練問你：「這篇想連結專業嗎？」 |
| 最終內容 | AI 產出 → 你修改 | 你的想法 → AI 潤飾 |

### 1.2 兩種模式的定位

升級後的發文工作室將提供兩種模式，分別針對不同程度的創作者：

**引導模式（教練模式）**：適合新手或需要引導的創作者。AI 教練會透過一系列的問題和建議，引導用戶完成創作。這個模式的特點是「教練引導多、用戶決策多」。

**進階模式（助手模式）**：適合成熟的創作者。用戶可以直接輸入素材，AI 作為助手提供切角分析和潤飾建議，但不會過度介入。這個模式的特點是「教練引導少、用戶自主多」。

### 2.2 調整項目 A2：合併「靈感工作庫」與「選題」步驟

**調整內容**：解決目前「靈感工作庫」與「選擇主題」重複選題的問題，將兩者合併為單一流程。

**調整方式**：

將現有的 Step 0（靈感狀態選擇）和 Step 2（選擇主題）合併為新的 Step 1（選題）。用戶可以選擇「自己有靈感」或「讓 AI 給選題」，如果選擇後者，AI 會直接生成選題，不再需要額外的選題步驟。

**新流程**：

```
Step 0: 創作意圖確認（純粹分享/順便帶專業/推廣專業）
Step 1: 選題（有靈感直接輸入 / 沒靈感讓 AI 給選題）
Step 2: 選擇貼文類型
Step 3: AI 教練問答（一次性問答）
Step 4: 選擇開頭風格（3-5 個選項）
Step 5: AI 生成內容
Step 6: 對話修改
Step 7: 人味潤飾
Step 8: 導向草稿庫
```

## 四、提示詞系統重構

### 4.1 提示詞模式開關

**調整內容**：建立 `buildPromptByMode` 函數，根據創作意圖決定注入哪些資料。

**程式碼設計**：

```typescript
// server/prompt-builder.ts

type CreativeIntent = 'pure_story' | 'light_connect' | 'full_professional';
type WritingMode = 'guided' | 'advanced';

interface PromptConfig {
  injectIpBase: boolean;
  injectViralFormula: boolean;
  injectFewShot: boolean;
  injectSuccessFactors: boolean;
}

const PROMPT_CONFIGS: Record<CreativeIntent, PromptConfig> = {
  pure_story: {
    injectIpBase: false,
    injectViralFormula: false,
    injectFewShot: false,
    injectSuccessFactors: false,
  },
  light_connect: {
    injectIpBase: true,
    injectViralFormula: false,
    injectFewShot: false,
    injectSuccessFactors: false,
  },
  full_professional: {
    injectIpBase: true,
    injectViralFormula: true,
    injectFewShot: true,
    injectSuccessFactors: true,
  },
};

export function buildPromptByMode(
  intent: CreativeIntent,
  mode: WritingMode,
  context: PromptContext
): string {
  const config = PROMPT_CONFIGS[intent];
  
  let prompt = buildBasePrompt(context);
  
  if (config.injectIpBase) {
    prompt += buildIpContext(context.ipProfile);
  }
  
  if (config.injectViralFormula) {
    prompt += buildViralFormulaContext(context.contentType);
  }
  
  if (config.injectFewShot) {
    prompt += buildFewShotContext(context.topic);
  }
  
  if (config.injectSuccessFactors) {
    prompt += buildSuccessFactorsContext(context.contentType);
  }
  
  return prompt;
}
```

### 5.2 「純粹分享」模式的資料引用

在「純粹分享」模式下，系統只引用以下資料：

**引用的資料**：
- 用戶輸入的素材（主題、故事細節、情緒）
- 用戶的口頭禪（在人味潤飾階段）
- 用戶的說話風格（作為語氣參考）

**不引用的資料**：
- IP 地基的所有欄位
- 爆款公式的所有欄位
- Few-Shot 範例
- 成功因素分析

**設計理由**：

這個模式的目標是讓用戶可以「純粹分享」，不被系統強制加入專業內容。因此，系統只保留最基本的寫作輔助（語氣、口頭禪），不注入任何可能影響內容方向的資料。

---

### 5.3 「順便帶專業」模式的資料引用

在「順便帶專業」模式下，系統引用以下資料：

**引用的資料**：
- 用戶輸入的素材
- IP 地基：職業/身份、人設三支柱（作為「專業連結建議」的參考）
- 用戶的口頭禪和說話風格

**不引用的資料**：
- IP 地基：核心信念、英雄旅程
- 爆款公式的所有欄位
- Few-Shot 範例
- 成功因素分析

**設計理由**：

這個模式的目標是讓用戶可以「自然地」連結專業，但不強制。系統會在生成內容後，根據 IP 地基提供「專業連結建議」，讓用戶自己決定要不要採用。

---

### 5.4 「推廣專業」模式的資料引用

在「推廣專業」模式下，系統引用以下資料：

**引用的資料**：
- 用戶輸入的素材
- IP 地基：所有欄位
- 爆款公式：成功因素、Hook 策略
- Few-Shot 範例：根據主題匹配的範例
- 用戶的口頭禪和說話風格

**設計理由**：

這個模式的目標是讓用戶可以「有效地」推廣專業或產品。系統會完整注入 IP 地基和爆款公式，幫助用戶產出高轉換的內容。

## 結論

本報告詳細說明了發文工作室從「代筆模式」轉型為「教練模式」的具體實作方案。主要的調整包括：

1. **新增創作意圖確認步驟**：讓用戶自己決定這篇貼文要不要連結專業
2. **合併靈感工作庫與選題步驟**：解決重複選題的問題
3. **AI 教練問答優化**：改為一次性問答，並調整為專業教練口語
4. **新增專業連結建議區塊**：讓用戶可以選擇是否採用建議
5. **進階模式轉型為助手模式**：減少教練引導，增加用戶自主性
6. **提示詞系統重構**：根據創作意圖動態調整資料注入

這些調整的核心目標是：**尊重創作者的意圖，讓 AI 成為教練而非代筆者**。
