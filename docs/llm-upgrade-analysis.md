# LLM 模型升級評估報告

## 一、四個階段測試的詳細設定

### 測試設定說明

四個成長階段的測試**沒有**使用任何語言風格或戰略分析中的爆款文案。測試腳本使用的是**純模擬資料**，目的是驗證「自適應品質門檻」和「成長階段判斷」的邏輯是否正確。

### 各階段模擬資料

| 階段 | 貼文數 | 貼文內容（模擬） | 互動範圍 |
|------|--------|------------------|----------|
| **新手期** | 3 篇 | 「今天開始經營自媒體」「分享一個小技巧」「第一次嘗試寫長文」 | 18-31 |
| **成長期** | 5 篇 | 「這個方法幫我省了很多時間」「分享我的創業心得」等 | 100-255 |
| **成熟期** | 10 篇 | 「五年創業經驗總結」「這個觀點可能會顛覆你的認知」等 | 237-540 |
| **專家級** | 20 篇 | 「這篇文章改變了十萬人」「我的方法論完整公開」等 | 884-1950 |

### 開頭生成的 Prompt 設定

測試中使用的 Prompt 是**簡化版本**，僅包含：
- 用戶階段資訊
- 系統/用戶範例權重
- 基本的開頭生成指令

**沒有**包含：
- 用戶的語言風格設定
- 戰略分析中的爆款文案
- IP 地基資料

### 結論

測試結果顯示的文案差異，主要來自於：
1. **系統/用戶範例權重不同**：新手 70% 系統範例 vs 專家 90% 用戶範例
2. **Prompt 中的階段描述**：告訴 LLM 用戶處於哪個階段
3. **LLM 的理解和生成**：根據階段資訊調整風格

如果要看到更明顯的差異，需要在實際系統中測試，因為實際系統會讀取用戶的：
- 語言風格設定
- 戰略分析中的爆款文案
- IP 地基資料
- 樣本貼文

## 四、分級使用策略評估

### 建議的分級策略

| 功能 | 建議模型 | 原因 |
|------|----------|------|
| 腦力激盪 | Flash | 創意發散，不需要高精度 |
| 開頭生成 | Flash | 生成多個候選，用戶選擇 |
| **正文生成** | **Pro** | 品質關鍵，需要高精度 |
| 品質檢查 | Flash | 規則驅動，不需要高精度 |

### 分級策略的成本影響

假設只有正文生成使用 Pro 模型（約佔 40% 的 Token 消耗）：

| 策略 | 計算 | 月度成本 |
|------|------|----------|
| 全部 Flash | 100% Flash | $33.60 |
| 分級使用 | 60% Flash + 40% Pro | $75.00 |
| 全部 Pro | 100% Pro | $137.00 |

**分級使用可節省約 45% 成本**，同時在關鍵環節（正文生成）獲得更高品質。

### 實作複雜度

**不會讓後台變麻煩**。實作方式：

```typescript
// 在 invokeLLM 中新增 model 參數
export async function invokeLLM(params: InvokeParams & { model?: 'flash' | 'pro' }) {
  const modelName = params.model === 'pro' 
    ? 'gemini-2.5-pro' 
    : 'gemini-2.5-flash-preview-09-2025';
  
  // 其餘程式碼不變
}

// 使用時
await invokeLLM({ messages, model: 'pro' }); // 正文生成
await invokeLLM({ messages }); // 預設使用 Flash
```

只需要：
1. 修改 `llm.ts` 新增 `model` 參數
2. 在正文生成的調用處加上 `model: 'pro'`

## 五、升級建議

### 方案比較

| 方案 | 月度成本 | 品質提升 | 實作複雜度 |
|------|----------|----------|------------|
| 維持現狀（Flash） | $33.60 | 基準線 | 無 |
| 全面升級 Pro | $137.00 | +30% | 低 |
| **分級使用（推薦）** | $75.00 | +20% | 低 |
| 升級 Gemini 3 Flash | $47.60 | +15% | 低 |

### 推薦方案：分級使用

1. **腦力激盪、開頭生成、品質檢查**：維持 Gemini 2.5 Flash
2. **正文生成**：升級到 Gemini 2.5 Pro

**預期效益**：
- 成本增加：$33.60 → $75.00（+123%）
- 正文品質提升：約 20-30%
- 指令遵循度提升：約 15%

### 實作步驟

1. 修改 `server/_core/llm.ts` 新增 `model` 參數
2. 在 `routers.ts` 的正文生成處使用 `model: 'pro'`
3. 測試驗證品質提升
4. 監控成本變化
