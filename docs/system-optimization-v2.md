## 二、優化影響範圍分析

### 2.1 發文工作室重構（影響最大）

#### 目前流程
```
選模式 → 輸入素材 → 選切角 → AI 直接生成完整文案 → 對話修改 → 儲存草稿
```

#### 新流程
```
選題（痛點矩陣/自訂）→ 素材收集（AI 問問題）→ 開頭選擇（3種風格）→ AI 生成 → 潤稿優化 → 生成串文 → 儲存草稿
```

#### 受影響的檔案
| 檔案 | 修改內容 |
|------|----------|
| `client/src/pages/WritingStudio.tsx` | 完全重構，改為多步驟引導式流程 |
| `server/routers.ts` | 修改 `ai.generateDraft`、`ai.analyzeAngles`，新增 `ai.collectMaterial`、`ai.generateHookOptions` |
| `shared/knowledge-base.ts` | 新增爆款文案結構模式、禁止詞彙列表 |

#### 潛在問題
1. **localStorage 狀態結構改變**：現有用戶的 localStorage 可能與新結構不相容
   - 解決：在頁面載入時檢測並清除舊格式
2. **草稿結構改變**：新增「選題來源」「使用的開頭風格」等欄位
   - 解決：資料庫 migration 新增欄位，舊草稿保持相容

## 九、確認事項

在開始實作前，請確認以下事項：

1. **發文工作室新流程**：
   - 選題 → 素材收集 → 開頭選擇 → AI 生成 → 潤稿 → 串文 → 儲存
   - 這個流程是否符合您的預期？

2. **AI 角色定位**：
   - AI 是「創作助手」而非「代筆者」
   - AI 會問問題收集素材，但最終生成完整內容
   - 這個定位是否正確？

3. **痛點矩陣格式**：
   - 生成「受眾會說的一句話」（用引號包起來）
   - 範例：「貓咪一直亂尿床，是不是討厭我？」
   - 這個格式是否符合您的預期？

4. **IP 地基使用方式**：
   - 作為「風格指南」確保語氣和視角一致
   - 不強制每篇文案都包含英雄旅程
   - 但 AI 可以建議「這個主題可以用你的 XX 經歷」

## 十、爆款文案結構模式（新增到知識庫）

根據分析的爆款文案集，整理出以下結構模式：

### 10.1 四種爆款類型

| 類型 | 特點 | 開頭範例 |
|------|------|----------|
| **清單型** | 標題點出數量，每點是獨立金句 | 「你開始轉運的五個徵兆」 |
| **故事型** | 以「我」開頭，有具體時間人物場景 | 「昨天我朋友跟我說...」 |
| **提醒型** | 直接對讀者說話，帶預言/祝福性質 | 「滑到這篇，表示好消息已經在路上」 |
| **觀點型** | 開頭就是核心觀點，後面用經歷支撐 | 「成長最快的方式，就是『硬著頭皮上』」 |

### 10.2 爆款共同特點

**開頭**：直接切入、不鋪墊、製造懸念或引發共鳴
**中間**：用轉折和因果推動情緒，有具體細節
**結尾**：互動問題或行動呼籲（「你可以留下一句『如願』」）
**語言**：口語化、用「我」「你」、短句為主、一句一行

### 10.3 禁止詞彙

- 開頭禁止：「讓我們」「一起來」「今天要分享」「不妨」「其實」
- 內容禁止：過多專業術語、長段落、說教語氣
- 格式禁止：Markdown 符號（**、#、`）

### 10.4 Threads 風格指南

1. **五年級可讀性**：文案要簡單到五年級小孩都能懂
2. **一句一行**：每個句子獨立成段，方便閱讀
3. **情緒推動**：用轉折、對比、懸念推動讀者情緒
4. **真人說話**：像朋友聊天，不像品牌廣告
5. **互動結尾**：用問題或行動呼籲引發互動
