## 五、實作建議

### 技術實作

目前系統使用統一的 `invokeLLM` 函數，只需新增 `model` 參數即可支援多模型：

```typescript
// 現有調用方式
const response = await invokeLLM({
  messages: [...],
});

// 新增 model 參數
const response = await invokeLLM({
  messages: [...],
  model: 'claude-sonnet-4', // 或 'gemini-2.5-flash', 'gpt-4o' 等
});
```

### 實作步驟

1. **Phase 1**：在 `llm.ts` 新增 model 參數支援
2. **Phase 2**：建立模型配置檔，定義各功能使用的模型
3. **Phase 3**：更新各功能調用處，傳入對應的 model 參數
4. **Phase 4**：新增 Feature Flag 控制，方便 A/B 測試

### 不會讓後台變麻煩

- 所有模型調用仍通過統一的 `invokeLLM` 函數
- 只需在配置檔中定義各功能使用的模型
- 可隨時通過 Feature Flag 切換模型

## 六、成本效益分析

### 假設條件
- 100 活躍用戶
- 每用戶每月生成 30 篇貼文
- 每篇貼文：1 次腦力激盪 + 5 個開頭 + 1 次正文 + 1 次品質檢查

### 成本比較

| 方案 | 月度成本 | 品質預期 | 成本增加 |
|------|----------|----------|----------|
| 目前（全 Flash） | $33.60 | 基準線 | - |
| 方案 B（成本優先） | $50-70 | +10-15% | +50-100% |
| **方案 A（推薦）** | **$85-120** | **+25-35%** | **+150-250%** |
| 方案 C（極致品質） | $180-250 | +40-50% | +400-600% |

### 投資回報

- 品質提升 25-35% 可能帶來更高的用戶滿意度和續約率
- 每月增加 $50-90 的成本，換取顯著的品質提升
- 建議從方案 A 開始，根據用戶反饋調整

## 八、附錄：模型 API 端點

### Manus Forge API 支援的模型

目前系統通過 Manus Forge API 調用 LLM，支援的模型包括：
- Gemini 系列（2.5 Flash, 2.5 Pro, 3 Flash, 3 Pro）
- OpenAI 系列（GPT-4o, GPT-4.1, GPT-5）
- Claude 系列（Sonnet 4, Opus 4.5）

### 切換模型的方式

在 `invokeLLM` 調用時傳入 `model` 參數即可切換模型，無需更改 API 端點或金鑰。
